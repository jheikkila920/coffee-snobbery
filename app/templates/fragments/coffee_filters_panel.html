{# Coffee filter dropdown panel (CAT-07 / 04-RESEARCH Open Question 2). Same shape as the inline filter form on `pages/coffees.html` but rendered as a swappable fragment for HTMX-fetched refresh — useful when a new roaster is created via the mini-modal and the filter dropdowns need updating without a full page reload. Context contract (passed by GET /coffees/filters-panel): - roasters: list[Roaster] — active roasters only. - countries: list[str] — distinct non-null countries. - processes: tuple[str, ...] — locked 6-value enum. - filters: dict — echoed back so the swap preserves selection (kept for symmetry; the refresh use case may pass {roaster_id: None, ...} for a clean reset). #}
