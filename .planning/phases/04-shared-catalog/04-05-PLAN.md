Endpoints:

1. `GET /` → list page (full page on first hit; `fragments/flavor_note_list.html` on HX-Request).

3. `POST /` → validate via `FlavorNoteCreate`; on ValidationError re-render form at 200; on success either `fragments/flavor_note_row.html` (normal path) OR empty body + `HX-Trigger: flavor-note-created` header (modal path, payload `{"flavor_note_id": fn.id, "name": fn.name}`).
4. `GET /{flavor_note_id}/edit` → form fragment pre-populated.
5. `POST /{flavor_note_id}` → update; same validation re-render shape.
6. `POST /{flavor_note_id}/archive` → archive; return updated row fragment.
7. `GET /datalist` (NOT `/list` — UI-SPEC and CONTEXT both use `/flavor-notes/datalist` as the endpoint name): same shape as the roasters autocomplete. If `len(q) < 2`: empty body. Form params on POST `/`: `name: str = Form(...)`, `category: str = Form(...)`, `as_modal: bool = Form(False)`.
