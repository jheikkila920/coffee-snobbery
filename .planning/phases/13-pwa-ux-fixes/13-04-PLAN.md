<objective>
Fix C6 + C7.

C6: The guided-brew "Audio & haptic cues" controls use `role="switch"` toggle pills (start screen) and emoji mute buttons (in-brew) that confuse users. Redesign both as clearly-labeled on/off controls. The localStorage cue prefs (`snobbery:gbm:cues`) and the `toggleChime()`/`toggleVibrate()` handlers in `guided-brew-mode.js` are PRESERVED unchanged — this is a template-only change.

C7a: The brew-log ratio readout (`brewRatio` Alpine scope) only updates via `x-on:input` (user typing). When a recipe/coffee selection programmatically prefills dose & water (the `/brew/prefill` HTMX swap into `#brew-prefill-region`, which is INSIDE the `brewRatio` scope), the new input values don't reach `brewRatio.dose`/`.water`, so the ratio is stale. Fix: re-sync the scope from the swapped input values via an Alpine `x-init` on the prefill region (or an `htmx:afterSwap`-driven call to `setDose`/`setWater`).

C7b: The rating star group wraps 4+1 at 375px because of `flex flex-wrap` on the star row (brew_form.html line 118). Five 44px star zones = 220px, which fits 375px — switch to `flex-nowrap` (and ensure star zones don't force-wrap).

Purpose: Clear cue controls + correct live ratio on prefill + single-line rating at 375px.
Output: Redesigned cue control markup, a prefill-region re-sync, and a non-wrapping star row.
</objective>
