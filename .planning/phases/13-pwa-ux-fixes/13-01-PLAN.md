<objective>
Fix C9, the LOAD-BEARING GATE for the whole phase: the service-worker cache name must bump on every `docker compose build` so a rebuild reaches installed PWAs without a manual "Clear site data." Without this, none of C1-C8/C10 are visible to John's installed iOS PWA.

Root cause (verified, RESEARCH §C9 / Pattern 5): `pwa.py:_get_build_hash()` derives the SW cache key from the hashed Tailwind CSS *filename*, which only changes when `tailwind.src.css` content changes. A rebuild that only touches a template or a Python route produces an identical CSS hash → identical `CACHE_NAME` → the new SW's activate handler sees no old caches to purge → stale shell on installed PWAs.

Purpose: Make the SW cache key change unconditionally per build via a `build_id.txt` artifact written in the Dockerfile.
Output: Modified `Dockerfile` (writes + copies `build_id.txt`), modified `pwa.py` (`_get_build_hash()` prefers `build_id.txt`), extended `tests/test_pwa.py` (versioning regression test).
</objective>
