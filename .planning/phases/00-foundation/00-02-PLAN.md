<objective>
Wire `app/logging.py` as the structlog ProcessorFormatter integration that emits one consistent JSON line per log event from stdout — across structlog calls, stdlib `logging`, uvicorn (error + access), FastAPI, and SQLAlchemy. The format is env-var controlled per CONTEXT D-16 (`LOG_FORMAT=json` default, `LOG_FORMAT=console` for color-on-TTY).

Purpose: FOUND-11 demands structured logging with JSON output and request correlation IDs. The correlation IDs themselves are minted by a Phase 1 middleware, but the structlog seat that picks them up via `contextvars.merge_contextvars` is laid down now so Phase 1 has nothing to retrofit. The pattern published here is what every later phase logs through — auth events (Phase 2), AI-call cost telemetry (Phase 7), scheduler summaries (Phase 8) — and the Phase 12 audit grep for "no PII / no request bodies in logs" is built against this configuration.

Output: One module (`app/logging.py`) exporting `configure_logging(format: str, level: str) -> None`; one test file asserting the JSON shape contains `event`, `timestamp_iso`, and `level` keys.
</objective>
