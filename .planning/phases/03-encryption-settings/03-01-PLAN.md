## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-T7 | Tampering / Spoofing | `api_credentials.provider` column accepting arbitrary values | mitigate | `CHECK (provider IN ('anthropic', 'openai'))` constraint at the DB layer (D-04, D-discretion default — Text + CHECK over Postgres ENUM); migration enforces canonical seeded set; Plan 03-04's `Provider = Literal[...]` type alias mirrors the constraint at the type-checker layer |
| T-03-T2-schema | Information disclosure | `api_credentials.key_ciphertext` column type drift (text vs bytea) leading to decode round-trips that could log/leak the encoded form | mitigate | Column is `LargeBinary` (`bytea`); model is `Mapped[bytes \| None]` (D-discretion locked); migration uses `sa.LargeBinary`, never `sa.Text` — keeps Fernet's `bytes` contract honest end-to-end |
</threat_model>
