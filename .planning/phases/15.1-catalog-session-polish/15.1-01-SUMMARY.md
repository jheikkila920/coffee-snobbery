# Phase 15.1 Plan 01: Multi-Origin Catalog Refactor Summary

**One-liner:** Replaced `coffees.country` + `coffees.origin` free-text columns with a structured `coffee_origins` join table (FK CASCADE, sort_order), updated all form/list/detail templates to render origins from the new relationship, and added HTMX server-side row-append for building multi-origin blends.

## What Was Built

**Schema migration (`p15_1_multi_origin`):** Creates `coffee_origins(id, coffee_id FK CASCADE, country TEXT NOT NULL, region TEXT NULL, sort_order INT NOT NULL DEFAULT 0)` with index `ix_coffee_origins_coffee_id`. Data-move INSERT uses `COALESCE(NULLIF(coffees.country,''), NULLIF(coffees.origin,''), 'Unknown')` to migrate every pre-existing coffee. Drops `coffees.country` and `coffees.origin` in the same transaction. Forward-only downgrade (no-op SELECT 1).

**CoffeeOrigin model:** `app/models/coffee_origin.py` — BigInteger Identity PK, coffee_id FK with ondelete=CASCADE, country/region/sort_order columns matching the migration shape exactly. Registered in `app/models/__init__.py` for Alembic autogenerate.

**Coffee model update:** Added `origins: Mapped[list[CoffeeOrigin]] = relationship("CoffeeOrigin", cascade="all, delete-orphan", order_by="CoffeeOrigin.sort_order")`. Removed `country` and `origin` `Mapped[str | None]` column declarations. `varietal` left for Plan 03.

**Schema (Pydantic):** Removed `country`, `origin`, `varietal` from `CoffeeCreate`. Origins bypass Pydantic via `getlist("origins_country")` / `getlist("origins_region")` in `_parse_form_payload` — same pattern as `advertised_flavor_note_ids`. `extra="forbid"` preserved; origin keys stripped before Pydantic sees the payload via `_ORIGIN_KEYS` set.

**Router changes:**
- `_parse_form_payload` returns 3-tuple `(raw_view, schema_input, origins)`; origins are `[(country, region|None), ...]` with blank-country rows stripped
- `_hydrate_form_context` accepts `origins` kwarg; defaults to `[("", None)]` for create; builds from `coffee.origins` relationship for edit
- `GET /coffees/origin-row-template`: new auth-gated endpoint returning `coffee_origin_row.html` fragment with `removable=True`
- Zero-origins guard: re-renders form with `errors["origins"]` message if all submitted countries are blank (T-15.1-03)
- `list_distinct_countries` query updated to `SELECT DISTINCT country FROM coffee_origins`

**Service changes:** `create_coffee` / `update_coffee` accept `origins: list[tuple[str, str | None]] | None`. Create: flush → insert CoffeeOrigins in sort_order sequence → commit. Update: delete-then-insert within same transaction.

**Templates:** `coffee_origin_row.html` new fragment with country/region inputs and HTMX 2.x `hx-on:click` remove button (CSP-safe, no inline `onclick`). `coffee_form.html` replaced standalone country+origin inputs with `#origin-rows-container` + Jinja include loop + "Add another origin" button (`hx-get=/coffees/origin-row-template hx-swap=beforeend`). `coffee_row.html`, `coffee_detail.html`, `coffee_filters_panel.html`, `pages/coffees.html` all updated to render from `coffee.origins` relationship.

**Tests:** 12 Wave 0 stubs in `tests/test_coffee_origins.py` covering table existence, data migration semantics, ORM blend derivation, cascade delete, and router integration. 6 existing test files updated to use new `create_coffee` signature and `origins_country` form field name.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical] TYPE_CHECKING guard for CoffeeOrigin in coffee.py**
- **Found during:** Task 5 ruff check
- **Issue:** `Mapped[list["CoffeeOrigin"]]` caused UP037 (remove quotes) + F821 (undefined name) — CoffeeOrigin is not imported at module level to avoid circular import, so the quoted forward reference was needed historically but is redundant with `from __future__ import annotations`. Correct fix is TYPE_CHECKING block.
- **Fix:** Added `from typing import TYPE_CHECKING` + `if TYPE_CHECKING: from app.models.coffee_origin import CoffeeOrigin`; removed quotes from annotation
- **Files modified:** app/models/coffee.py

**2. [Rule 1 - Bug] B905 zip() without strict= parameter**
- **Found during:** Task 5 ruff check
- **Issue:** `zip(origins_country_raw, origins_region_raw)` flagged by B905 (flake8-bugbear)
- **Fix:** Added `strict=False` to both zip calls in `_parse_form_payload` — semantics intentionally truncate to shortest list (form fields are always paired)
- **Files modified:** app/routers/coffees.py

**3. [Rule 1 - Bug] test_coffee_origins.py unused imports and redundant quotes**
- **Found during:** Task 5 ruff check
- **Issue:** `test_is_blend_derived_single` had unused `func`, `select`, `text` imports; `test_cascade_delete_removes_origins` had unused `text`, `engine`; all 4 router test function signatures had quoted `"Any"` type annotations
- **Fix:** Applied `ruff check --fix` auto-fix (I001 import order, F401 unused imports, UP037 remove quotes)
- **Files modified:** tests/test_coffee_origins.py

**4. [Rule 1 - Bug] Pre-existing tests calling create_coffee with old signature**
- **Found during:** Task 5 audit (scope: pre-existing test files that directly called the service)
- **Issue:** 4 test files passed `country=`, `origin=`, `varietal=` to `create_coffee` / used `"country"` as a POST form field; these would fail with TypeError since those params no longer exist
- **Fix:** Updated `_seed_coffee` helpers in `test_coffee_filters.py`, `test_routers_bags.py`, `test_routers_coffees.py` to use `origins=[(country, None)] if country else []`; updated form-post tests in `test_routers_coffees.py`, `test_equipment_create_fragment.py`, `test_happy_path_smoke.py` to use `origins_country=` / `origins_region=` keys
