---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | pytest 9.x (+ pytest-asyncio, respx) |
| **Config file** | `pyproject.toml` (pytest section); fixtures in `tests/conftest.py` |
| **Quick run command** | `docker compose exec coffee-snobbery python -m pytest -q tests/{file}` |
| **Full suite command** | `docker compose exec coffee-snobbery python -m pytest -q` |
| **Estimated runtime** | ~120s full suite (940 tests at Phase 15) |

**Container test gate (baked image, no source bind-mount):**
- Rebuild + restart before each run: `docker compose build coffee-snobbery && docker compose up -d coffee-snobbery`
- OR fast iteration: `docker compose cp tests/ coffee-snobbery:/app/tests/` (file-level, not dir-level — nesting trap; see memory entry)
- pytest is NOT in production image: `pip install --user pytest pytest-asyncio respx` after every rebuild

---

## Sampling Rate

- **After every task commit:** Run targeted `pytest -q tests/test_{area}.py` (CATALOG-NN scope)
- **After every plan wave:** Run full suite `python -m pytest -q`
- **Before `/gsd-verify-work`:** Full suite must be green (drop snobbery_test DB before full run; see memory entry)
- **Max feedback latency:** 30s for targeted tests, 120s for full suite
