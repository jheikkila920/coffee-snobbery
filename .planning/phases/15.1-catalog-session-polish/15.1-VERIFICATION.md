## Human Verification Required

### 1. Desktop Edit Form Width Parity (CATALOG-01)

**Test:** Open `/coffees`, `/roasters`, `/equipment`, `/flavor-notes`, `/recipes` in Chrome, Firefox, and Edge at viewport >=768px. Click Edit on any row.
**Expected:** The edit form renders inside the `#{entity}-form-mount` div at full table width — not crammed into the row's action column. Form is visually identical to the create form in terms of width.
**Why human:** CSS rendering of the OOB-swapped fragment width depends on browser layout engine behavior; static code shows the `#entity-form-mount` wiring is correct but visual parity requires a real browser.

### 2. Mobile Inline Edit (CATALOG-01 D-21)

**Test:** On an iPhone in the PWA-installed app at <768px viewport, tap Edit on any catalog row.
**Expected:** The edit form appears inline, replacing the row via `closest [data-row]` outerHTML swap — the `md:hidden` mobile button behavior unchanged from commit a3a2f76.
**Why human:** PWA-installed tactile behavior on physical device; not reproducible in Playwright without a device-mode browser.

### 3. Silent Re-click Overwrites (CATALOG-01 D-15)

**Test:** Click Edit on row A to open a form; while the form is visible, click Edit on row B.
**Expected:** Row B's form silently replaces row A's form in the mount div. No dirty-state prompt.
**Why human:** Interactive UX flow requiring a real browser session with an open form state.

### 4. Home Page Has No Freshness Card (CATALOG-07)

**Test:** Open `/` in a browser that has cleared site data (not just bypassed service worker). Inspect the page.
**Expected:** No roast-freshness section anywhere on the home page. The `/home/cards/roast-freshness` endpoint returns 404.
**Why human:** Service-worker stale cache can serve old pages after a rebuild; "Clear site data" is required to confirm the template change reached the browser.

### 5. Varietal Autocomplete UX (CATALOG-05 D-03)

**Test:** Open the coffee create or edit form. Type "Gei" in the varietal field. Then type a name that doesn't exist ("FancyCustom").
**Expected:** "Geisha" appears in the autocomplete dropdown after typing "Gei". After typing "FancyCustom", an "Add 'FancyCustom' as a new varietal?" affordance appears. Clicking it creates the varietal and renders it as a chip.
**Why human:** HTMX debouncing with real keystrokes; chip-builder fragment rendering requires a live browser session.

### 6. Cross-User Bidirectional Flavor Sync (CATALOG-06 D-08/D-09)

**Test:** In two browser sessions (User A and User B), both navigate to the same coffee. User A saves a brew session adding flavor note "Jasmine" not currently on the coffee. User B starts a new brew session for the same coffee.
**Expected:** User B's prefill includes "Jasmine". Then User A saves a session removing "Jasmine" from the chips. User B's next new-session prefill no longer includes "Jasmine".
**Why human:** Multi-user concurrency requires two simultaneous authenticated browser sessions; not reproducible in single-user pytest without elaborate mocking.
