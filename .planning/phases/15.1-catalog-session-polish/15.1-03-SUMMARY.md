## What Was Built

### CATALOG-04: Roast Level Enum Widening

Migration `p15_1_roast_level_enum` (chained after `p15_1_drop_roast_date`) drops and recreates the `coffees_roast_level_check` CHECK constraint with 8 values. `COFFEE_ROAST_LEVELS` tuple updated to `("ultra-light", "nordic-light", "light", "medium-light", "medium", "medium-dark", "dark", "unknown")`. `CoffeeCreate.roast_level` regex updated to match. `Coffee` model CheckConstraint literal updated for Alembic drift prevention.

### CATALOG-05: Varietal M2M

Migration `p15_1_varietal_m2m` (chained after `p15_1_roast_level_enum`):
- Creates `varietals` table: `id BIGINT IDENTITY PK`, `name CITEXT UNIQUE NOT NULL`, `created_at TIMESTAMPTZ NOT NULL DEFAULT now()`
- Creates `coffee_varietals` join table: `coffee_id ON DELETE CASCADE`, `varietal_id ON DELETE RESTRICT`
- Seeds 14 varietals via `op.bulk_insert`: Bourbon, Typica, Caturra, Catuai, Geisha, Pacamara, SL28, SL34, Mundo Novo, Pacas, Heirloom, Maragogype, Castillo, Catimor
- Drops `coffees.varietal` text column

`Varietal` model in `app/models/varietal.py` with CITEXT unique name. `coffee_varietals` declared as SQLAlchemy `Table` (not ORM class). `Coffee.varietals` relationship uses `secondary="coffee_varietals"`, `lazy="selectin"`, `order_by="Varietal.name"`.

### Service Layer

`app/services/varietals.py` mirrors `flavor_notes.py` exactly:
- `create_varietal(db, *, name, by_user_id, commit=True)` — IntegrityError → DuplicateNameError pattern; structlog `CATALOG_VARIETAL_CREATED` audit event
- `search_by_prefix(db, *, query, limit=50)` — CITEXT ilike prefix match, no archived filter (Varietal has no archived column)

### Router Endpoints

Two new endpoints in `app/routers/coffees.py` declared before `/{coffee_id}` to avoid Starlette path-capture:
- `GET /coffees/varietal-autocomplete?q=<prefix>` — returns `varietal_autocomplete.html` fragment; empty response if `len(q) < 2` (T-15.1-19)
- `POST /coffees/varietals` — CSRF-protected create-on-the-fly; DuplicateNameError → re-queries existing varietal and returns it as chip (graceful concurrent UX)

`_parse_form_payload` extended to 4-tuple `(raw_view, schema_input, origins, varietal_ids)`. `varietal_ids` stripped from Pydantic payload via `_ORIGIN_KEYS`. `create_coffee` and `update_coffee` accept `varietal_ids: list[int] | None = None`.

### Frontend

`coffee_form.html`: single varietal text input replaced with `varietalChips` chip-builder (distinct Alpine scope from `flavorNoteChips`). Pattern: server-seed chips with `data-seed-chip`, hidden inputs `name="varietal_ids"`, `data-initial-chips='{{ selected_varietals|tojson }}'` (single-quoted per Pitfall 5), HTMX autocomplete to `/coffees/varietal-autocomplete`.

`coffee_detail.html`: `coffee.varietal` single-text replaced with `{% for v in coffee.varietals %}{{ v.name }}{% endfor %}` comma-separated list.
