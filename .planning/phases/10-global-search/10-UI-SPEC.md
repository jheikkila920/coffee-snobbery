## Spacing Scale

Declared values (multiples of 4 only). All tokens already in use across phases 1–9; Phase 10 inherits without change.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon-to-label gap inside the search icon button; gap between match-highlight spans |
| sm | 8px | Padding inside result row (vertical); gap between result context lines |
| md | 16px | Horizontal padding inside the search input; padding inside result rows (horizontal); section spacing in the mobile sheet |
| lg | 24px | Vertical padding of the persistent header bar; bottom padding of the dropdown panel |
| xl | 32px | Top padding inside the mobile full-screen sheet (below close button) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Search icon tap target: minimum 44×44px (MOB-04 hard rule). The icon button must have `min-h-[44px] min-w-[44px]`.
- Mobile sheet close (X) button: minimum 44×44px tap target.
- Result row minimum height: 44px (`min-h-[44px]`) so every result is thumb-operable.

**Source:** 8-point scale established Phase 0; tap-target rule from MOB-04 (REQUIREMENTS.md).

## Typography

All sizes and weights are already committed in prior phases. Phase 10 uses the same scale — no new type roles.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 16px | 400 (normal) | 1.5 | Result row entity name; snippet text; "+N more" hint; empty-state body |
| Label | 14px | 400 (normal) | 1.4 | Result context line (roaster, origin, date, snippet truncation); group header separator label; "Archived" badge text |
| Heading | 20px | 600 (semibold) | 1.2 | Group-header text ("Coffees", "Roasters", etc.); mobile sheet heading "Search" |
| Display | 24px (`text-2xl`) | 600 (semibold) | 1.2 | Not used in this phase |

**Match highlight (`<mark>` / `<strong>`):** `font-semibold` (weight 600) on the matched substring. No color change — weight alone is sufficient contrast and is safe without `|safe`. The mark element itself must not introduce a background color that breaks dark mode.

**Source:** Typography scale from prior phase UI-SPECs; committed values in tailwind.src.css and home.html pattern (`text-2xl font-semibold` headings, `text-base` body, `text-sm` secondary).

## Component Inventory

### 1. Persistent Search Header

**Location:** `base.html` — added between `<body>` open and `{% block content %}`, gated on `request.state.user`.

**Desktop (≥768px):**
- Thin horizontal bar, full-width, `bg-cream-100 dark:bg-espresso-900 border-b border-espresso-200 dark:border-espresso-700`
- Height: 56px (`h-14`), padded `px-6`
- Contains: inline text input (`w-64` or `w-80` at lg breakpoint), magnifying glass icon to the left of the input
- Input: `rounded border border-espresso-200 dark:border-espresso-700 bg-cream-50 dark:bg-espresso-900 px-3 py-2 text-base w-64 lg:w-80`
- `placeholder="Search coffees, roasters, recipes…"` (14-char truncation on narrow; full text at lg+)
- `hx-get="/search"` `hx-trigger="keyup changed delay:250ms"` `hx-sync="this:replace"` `hx-target="#search-results"` `hx-indicator="#search-spinner"`
- `name="q"` `minlength="2"` (client-side attribute, server ignores < 2 chars anyway)
- Results dropdown `id="search-results"` positioned absolute below the input, `z-50 w-full max-w-xl`

**Mobile (<768px):**
- Header shows only a magnifying glass icon button (`min-h-[44px] min-w-[44px]`)
- Icon: SVG magnifying glass, 20×20px, `text-espresso-700 dark:text-cream-100`
- Tap opens the full-screen sheet (Alpine `x-data="searchBar"`, `x-show="sheetOpen"`)
- Sheet: `fixed inset-0 z-50 flex flex-col bg-cream-50 dark:bg-espresso-950`
- Sheet header: close (X) button `min-h-[44px] min-w-[44px]` top-right, `px-4 pt-4 pb-2`
- Input: `w-full rounded border border-espresso-200 dark:border-espresso-700 bg-cream-50 dark:bg-espresso-900 px-3 py-3 text-base` — auto-focused on sheet open (`x-init="$el.querySelector('input').focus()"`)
- Results fill `flex-1 overflow-y-auto`
- Scrim: behind sheet, `bg-espresso-950/50 dark:bg-espresso-950/70`, tap dismisses
- Close affordances: X button, Esc key (Alpine `@keydown.escape.window="sheetOpen=false"`)
- No `history.pushState` — back button does not close the sheet (D-03)

### 2. Search Results Fragment (`#search-results`)

**Rendered by:** `GET /search?q={query}` → `fragments/search_results.html`
**Cache headers:** `Cache-Control: no-store` + `Vary: HX-Request` (inherited from fragment_cache middleware)

**Structure (when results exist):**

```
[Group header: "Coffees" (text-lg font-semibold, group-header divider bg)]
  [Result row] [Result row] ... (max 5)
  [+N more — keep typing to narrow]  ← non-clickable, text-sm text-espresso-600

[Group header: "Roasters"]
  ...

[Group header: "Recipes"]
  ...

[Group header: "Equipment"]
  ...

[Group header: "Flavor Notes"]
  ...

[Group header: "Your Brew Notes"]
  ...
```

**Fixed group order** (D-07): Coffees → Roasters → Recipes → Equipment → Flavor Notes → Your Brew Notes. Groups with zero matches are omitted entirely.

**Result row anatomy:**

| Entity | Primary text | Context text (text-sm text-espresso-600) | Link destination |
|--------|-------------|------------------------------------------|-----------------|
| Coffee | `name` | `roaster_name · origin` (each optional) | `/coffees/{id}` |
| Roaster | `name` | location (optional) | `/roasters/{id}/edit` |
| Recipe | `name` | `description` truncated to 60 chars | `/recipes/{id}/edit` |
| Equipment | `name` | type label | `/equipment/{id}/edit` |
| Flavor Note | `name` | category label | `/flavor-notes/{id}/edit` |
| Brew Note | `coffee_name` | `brewed_at date · snippet` (40-char window around match) | `/brew/{id}/edit` |

**Result row styling:**
```
class="block px-4 py-2 min-h-[44px] hover:bg-cream-200 dark:hover:bg-espresso-700
       focus:bg-cream-200 dark:focus:bg-espresso-700 outline-none"
```

**Match highlight (D-06):**
- Matched substring wrapped in `<strong class="font-semibold">` via `markupsafe.Markup` composition — never `|safe` on user text
- Surrounding text is autoescaped by Jinja as normal string fragments
- No background color on `<strong>` — weight 600 is the only visual distinction
- `<mark>` element may be used instead of `<strong>` with `class="bg-transparent font-semibold not-italic"` to preserve semantic HTML without injecting visual styles that break dark mode

**"+N more" hint:**
```
class="block px-4 py-2 text-sm text-espresso-600 dark:text-cream-300 cursor-default select-none"
```
Copy: `"+{N} more — keep typing to narrow"`

**Archived badge (D-12):**
```
class="ml-2 inline-block rounded-full bg-espresso-200 px-2 py-0.5 text-xs
       text-espresso-800 dark:bg-espresso-700 dark:text-cream-100"
```
Text: `"Archived"` — matches the existing pattern from coffee_row.html exactly.

**Group header:**
```
class="px-4 py-1 text-sm font-semibold text-espresso-700 dark:text-cream-300
       bg-cream-200 dark:bg-espresso-800 sticky top-0"
```
Sticky so the current group label stays visible while scrolling a long list.

**HTMX spinner:**
```html
<span id="search-spinner" class="htmx-indicator">
  <svg ...>  <!-- 16×16 spinning circle SVG, text-espresso-700 dark:text-cream-100 -->
</span>
```
The `.htmx-indicator` rule already exists in `tailwind.src.css` — no new CSS needed.

### 3. Alpine Component: `searchBar`

Registered in `app/static/js/alpine-components/search-bar.js`, loaded in `base.html` before the Alpine CSP core (same pattern as existing components).

**State:** `{ sheetOpen: false }`
**Methods:**
- `openSheet()` — sets `sheetOpen = true`
- `closeSheet()` — sets `sheetOpen = false`, clears input value, clears `#search-results`
- Esc handler: `@keydown.escape.window="closeSheet()"` — only when `sheetOpen`
- Backdrop click: `@click.self="closeSheet()"` on the scrim element

**CSP constraint:** No `eval`, no `new Function`. All handlers are declarative Alpine attributes or registered `Alpine.data()` factory. Matches existing component pattern exactly.

## Interaction Contract

### HTMX Wiring

| Attribute | Value |
|-----------|-------|
| `hx-get` | `/search` |
| `hx-trigger` | `keyup changed delay:250ms` |
| `hx-sync` | `this:replace` — cancels in-flight request when user keeps typing (HX-4) |
| `hx-target` | `#search-results` |
| `hx-indicator` | `#search-spinner` |
| `name` | `q` |
| `autocomplete` | `off` |
| `hx-swap` | `innerHTML` |

**Minimum length guard:** The server returns an empty 200 response (no HTML) for `q` with fewer than 2 characters. The client-side `minlength="2"` attribute is an advisory hint only — the server is authoritative.

**No CSRF token required:** `GET /search` is read-only. The fragment_cache middleware adds `Cache-Control: no-store` + `Vary: HX-Request` automatically.

### Enter Key / Arrow Key Behavior (Claude's Discretion)

Per D-04 (live results only, no results page), Enter is a no-op on the search input (`@keydown.enter.prevent=""`). No keyboard arrow navigation in v1 — deferred per CONTEXT.md.

### Result Link Navigation

All result links are standard `<a href="...">` — full-page navigations, not HTMX swaps (D-11). This matches the mobile back-button expectation: tapping a result closes the sheet implicitly via page navigation.

### Dropdown Dismiss (Desktop)

- Outside-click: `@click.outside="resultsOpen = false"` on the input wrapper
- Esc: clears `#search-results` contents and removes focus ring
- Navigating to a result link closes the dropdown by page navigation

### Reduced Motion

No animations beyond the `.htmx-indicator` opacity transition (150ms, already defined in `tailwind.src.css`). Mobile sheet does not use a slide-in animation — appears immediately. This satisfies `prefers-reduced-motion` without additional media queries.

### `aria-live` Region

The `#search-results` element carries `aria-live="polite"` and `aria-atomic="false"`. Group headers use `role="group"` with `aria-label="{group name}"`. Result links carry descriptive `aria-label` when the visible text alone is ambiguous (e.g. brew notes: `aria-label="Brew note: {coffee_name}, {date}"`).

```html
<div id="search-results"
     role="listbox"
     aria-label="Search results"
     aria-live="polite"
     aria-atomic="false">
  <!-- populated by HTMX -->
</div>
```
