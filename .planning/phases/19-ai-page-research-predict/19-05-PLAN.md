|----------|-------------|
| client POST → research / improve-brew | quota bypass + IDOR cross here |
| client GET → chart JSON | per-user data scoping boundary |
| SSE stream → NPM → browser | buffering + error-leak boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-16 | Elevation | research/improve-brew quota bypass via direct POST | mitigate | quota check keyed to request.state.user.id; DB-backed; checked BEFORE EventSourceResponse starts (V4) |
| T-19-17 | Elevation/IDOR | POST /ai/improve-brew/{session_id} cross-user | mitigate | session loaded by_user_id → 404 (existence non-leak, mirrors edit_brew_form, V4) |
| T-19-18 | Info Disclosure | chart JSON leaking other users' data | mitigate | both chart queries per-user scoped on user_id |
| T-19-19 | Info Disclosure | SSE event:error stack trace | mitigate | error event emits user-facing string; structlog logs the exception server-side (V7) |
| T-19-20 | Tampering | SSE buffered/replayed behind NPM | mitigate | X-Accel-Buffering:no header + Cache-Control:no-cache (sse-starlette) + NPM proxy_buffering off (documented in 19-07) |
| T-19-SC | Tampering | sse-starlette runtime use | mitigate | already approved in 19-01 audit; pinned >=3.4,<4 |
