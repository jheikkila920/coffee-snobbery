<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /csp-report | Browser-driven uncapped traffic; a single broken page can fire hundreds of violations per second |
| client → any HTTP endpoint | Must NOT be able to bypass CSP enforcement via header manipulation |
| upstream content (CDN scripts) → CSP | Even trusted CDN scripts must carry the per-request nonce; without it, all third-party scripts are blocked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | XSS via inline scripts or `'unsafe-eval'` | mitigate | `script-src 'self' 'nonce-{nonce}'` with NO `'unsafe-eval'` or `'unsafe-inline'`; per-request nonce makes the policy strict-dynamic-compatible (ASVS V13.1.1, V13.2.4 — Content Security Policy) |
| T-03-02 | Tampering | Clickjacking via iframe embedding | mitigate | `X-Frame-Options: DENY` + CSP `frame-ancestors 'none'` belt-and-braces (ASVS V13.2.5) |
| T-03-03 | Tampering | MIME-confusion attacks | mitigate | `X-Content-Type-Options: nosniff` on every response (ASVS V13.2.6) |
| T-03-04 | Information Disclosure | Referrer leakage to external sites | mitigate | `Referrer-Policy: strict-origin-when-cross-origin` (ASVS V8.2.1) |
| T-03-05 | Tampering / Information Disclosure | Feature abuse (camera, microphone, geolocation) by injected JS | mitigate | `Permissions-Policy: camera=(self), microphone=(), geolocation=(), ...` allowlists each browser-level capability (ASVS V13.2.7) |
| T-03-06 | Denial of Service | `/csp-report` flood from a misconfigured CSP firing on every page-load (pitfall 13.5) | mitigate | slowapi `30/minute` per-IP limit; handler does no DB writes — only one structlog call; malformed JSON swallowed without crash (ASVS V13.4.1) |
| T-03-07 | Information Disclosure | CSP report bodies containing inadvertent PII (Referer URL with tokens) | mitigate | `_log_csp_violation` strips report payload to four documented fields (`blocked_uri`, `violated_directive`, `line`, `source_file`) — drops everything else (ASVS V7.1.1) |
| T-03-08 | Spoofing | Forged `Reporting-Endpoints` accepted by browser | accept | `report-to csp-report` value `csp-report="/csp-report"` is same-origin only; impossible for an attacker to redirect reports off-origin without already controlling the response headers (which is the threat we're already mitigating) |
| T-03-09 | Tampering | csp_nonce missing on response (pitfall 13.4) | mitigate | Defensive fallback `CSP_FALLBACK` emits `script-src 'self'` (no nonce term) plus a `csp.nonce_missing` WARNING log — fails closed (blocks all inline scripts) rather than open |
</threat_model>
