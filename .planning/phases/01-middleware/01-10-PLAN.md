    Create `docs/decisions/0002-pure-asgi-middleware.md` using the standard ADR template:

    `# ADR 0002: All Custom Middleware Is Pure ASGI`

    Status: Accepted. Date: 2026-05-16. Phase: 1 (Middleware).

    `## Context`
    Starlette historically offered `BaseHTTPMiddleware` as an ergonomic way to write middleware as a class with a `dispatch(request, call_next)` method. This API has a known limitation: it breaks `contextvars.ContextVar` propagation across the middleware chain. AUTH-10 requires structured logging with `request_id` correlation; the correlation is bound to a contextvar. A `BaseHTTPMiddleware` inserted into the stack silently strips the `request_id` from logs emitted by inner middlewares or route handlers — a defect that is hard to diagnose because no error is raised.

    `## Decision`
    Every custom middleware in `app/middleware/` MUST be a **pure ASGI middleware** — a class with:

    `class M:\n    def __init__(self, app: ASGIApp, **opts) -> None: ...\n    async def __call__(self, scope: Scope, receive: Receive, send: Send) -> None: ...`

    `BaseHTTPMiddleware` is forbidden. Third-party middleware that uses `BaseHTTPMiddleware` (e.g., `starlette-csrf` — VERIFIED PURE ASGI as of 3.0) is allowed only if confirmed pure ASGI.

    `## Consequences`
    - All five Phase-1 middlewares (`RequestContextMiddleware`, `SecurityHeadersMiddleware`, `FragmentCacheHeadersMiddleware`, `SessionMiddleware`, and the third-party `CSRFMiddleware`) follow this rule.
    - Future contributors writing new middleware MUST follow the pure ASGI template from RESEARCH §4.
    - When wrapping outgoing responses, use the `send_wrapper(message)` pattern — intercept `http.response.start` to mutate headers; pass other message types through.
    - Performance benefit (incidental): pure ASGI middlewares show 20–30% lower per-request overhead than `BaseHTTPMiddleware` equivalents (RESEARCH §4 citation).

    `## Enforcement`
    - `app/middleware/__init__.py` carries a warning comment citing this ADR.
    - Phase 12 may add a `grep -r "BaseHTTPMiddleware" app/middleware/` CI check; until then, this ADR is enforced at PR review.

    `## Alternatives Considered`
    - **Allow `BaseHTTPMiddleware`** with documented contextvars limitations: rejected — the limitation is silent (no error raised) and would inevitably bite us.
    - **Use `asgi-correlation-id` library** for request_id: declined — pure ASGI hand-rolled implementation is ~30 LOC; adding a dependency for that footprint is overkill.

    `## References`
    - AUTH-10 (REQUIREMENTS.md)
    - RESEARCH.md §4, §13.1
    - https://www.starlette.io/middleware/
    - https://github.com/Kludex/starlette/discussions/2160
  </action>
