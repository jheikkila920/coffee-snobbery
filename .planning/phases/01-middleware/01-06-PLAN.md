<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → browser cache / intermediate proxies | Authenticated responses or HTMX fragments cached by browser/proxy could expose private data to a subsequent user on a shared device (back-button after logout shows cached page) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Authenticated full-page response cached and shown after logout via browser back-button | mitigate | `Cache-Control: private, no-cache, must-revalidate` on every non-HTMX response forces revalidation; bfcache is still allowed (the only way to break it would be `no-store`, which would also break legitimate fast back-button UX) (ASVS V8.3.1, V13.4.1) |
| T-06-02 | Information Disclosure | HTMX fragment containing user-specific data cached and served from disk on a subsequent navigation | mitigate | `Cache-Control: no-store` AND `Vary: HX-Request` on every HTMX response — `no-store` prevents disk cache; `Vary` ensures intermediate proxies don't serve the fragment to a non-HTMX request (D-11) (ASVS V13.4.1) |
| T-06-03 | Information Disclosure / Tampering | An intermediate proxy strips the response's own `Cache-Control` and caches per its default policy | accept | Route-set `Cache-Control` is preserved (D-12) — middleware does not overwrite. If a route ships a relaxed policy intentionally, that's the route's design choice; review at PR time. |
| T-06-04 | Tampering | A future route author forgets to set `Cache-Control` and relies on the middleware default — but uses `BaseHTTPMiddleware` for a custom transformer that breaks the middleware chain | mitigate | Module docstring + `app/middleware/__init__.py` warn against BaseHTTPMiddleware; Plan 10 ADR formalizes; pitfall 13.1 captured |
| T-06-05 | Information Disclosure | `/static/*` files (Tailwind, HTMX, Alpine) being served with `no-store` would break performance and could leak path info | mitigate | `static_prefixes=("/static/",)` bypasses the middleware entirely — StaticFiles handles its own headers (content-hashed filenames + immutable cache per FOUND-12). |
</threat_model>
