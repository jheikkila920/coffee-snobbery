<objective>
Implement the OUTERMOST middleware in the Phase 1 stack: assigns `request_id`, mints the `csp_nonce`, binds them to contextvars + `scope["state"]`, and finalizes structlog configuration so every log line (including foreign uvicorn/SQLAlchemy logs) flows through one JSON formatter with one redaction processor.

Purpose: Satisfies AUTH-10 (auth events logged with `user_id`, `ip`, `timestamp`; no PII or request bodies in logs) and provides the foundation for every other middleware's log correlation. This is the *first* middleware on the request path (last `add_middleware()` call in Plan 09) — every other layer depends on the `request_id` and `csp_nonce` it stamps on the scope.

Output: `app/middleware/request_context.py` + `app/logging_config.py` + `app/events.py`. Wave 0 tests `tests/middleware/test_logging.py::test_redaction_processor`, `::test_redaction`, `::test_contextvars_propagation` flip from skip/red to green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream → app | An untrusted client may attempt to inject log lines via crafted `X-Request-Id` headers (log forging) |
| inside app → log stream | Structured log lines may be inadvertently emitted with sensitive values (password, API key) by future code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | structlog event_dict carrying password/api_key/session_token/cookie/authorization/csrftoken | mitigate | `_redact_sensitive_fields` processor replaces those keys' values with `"***REDACTED***"` as the LAST shared step, before JSON renderer — defense in depth against future "I'll just log everything for debugging" mistakes (ASVS V7.1.1, V7.3.1) |
| T-02-02 | Tampering / Repudiation | `X-Request-Id` from untrusted clients used as the correlation ID | mitigate | Validate against `^[A-Za-z0-9_-]{1,128}$` regex before honoring; mint fresh `secrets.token_urlsafe(8)` if non-conforming. Prevents log injection (e.g., `X-Request-Id: \n{"event":"auth.login_succeeded"}` is rejected) (ASVS V7.3.4) |
| T-02-03 | Repudiation | contextvars leakage across requests (pitfall 13.2) | mitigate | `clear_contextvars()` called both at request entry AND in `finally` — prevents request_id of request N appearing in logs of request N+1 (data-integrity for audit logs) (ASVS V7.1.3) |
| T-02-04 | Tampering | future contributor writing a `BaseHTTPMiddleware` subclass that breaks contextvars (pitfall 13.1) | accept | Module docstring + `app/middleware/__init__.py` comment warn against it; Plan 10 ADR formalizes; Phase 12 may add a grep test if drift emerges |
| T-02-05 | Information Disclosure | uvicorn.access default formatter logs full URLs which may include tokens in query strings | mitigate | uvicorn.access is rerouted through structlog's ProcessorFormatter; the redaction processor is in the foreign_pre_chain so any sensitive-key fields are caught even from foreign loggers (ASVS V7.1.1) |
</threat_model>
