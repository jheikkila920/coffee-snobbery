```javascript
document.addEventListener('alpine:init', () => {
  Alpine.data('iosBanner', () => ({
    show: false,
    init() {
      if (localStorage.getItem('snobbery:ios-banner-dismissed')) return;
      const isIOS = /iPhone|iPad|iPod/.test(navigator.userAgent);
      const isStandalone = window.navigator.standalone === true ||
                           window.matchMedia('(display-mode: standalone)').matches;
      if (isIOS && !isStandalone) { this.show = true; }
    },
    dismiss() {
      this.show = false;
      try { localStorage.setItem('snobbery:ios-banner-dismissed', '1'); } catch (_e) {}
    },
  }));
});
```

- Rationale: respx is already in the test deps; monkeypatch sidesteps Fernet encryption round-trip noise; the test is about UI/template rendering on the basis of the helper's return value, NOT about whether the encryption layer works (that has its own coverage in tests/services/test_credentials.py). Symmetric across plans 17-03 and 17-04 so the same approach reads identically in both modules.
- Concrete signature: use pytest's `monkeypatch` fixture and a side-effect function:
  ```python
  def _with_ai_key(monkeypatch, *, provider: str = "anthropic"):
      from app.services import credentials as credentials_service
      def fake(_db, p):
          return object() if p == provider else None
      monkeypatch.setattr(credentials_service, "get_provider_credential", fake)

  def _without_ai_key(monkeypatch):
      from app.services import credentials as credentials_service
      monkeypatch.setattr(credentials_service, "get_provider_credential", lambda _db, _p: None)
  ```
