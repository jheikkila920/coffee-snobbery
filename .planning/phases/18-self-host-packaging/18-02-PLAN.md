<objective>
Prepare the Dockerfile and the in-app version display to consume a build-time `APP_VERSION` stamp so the release workflow (Plan 04) can pass `${{ github.ref_name }}` as a build-arg and have the running container report the git tag everywhere — `/admin` page, `docker inspect`, OCI labels. Bump `pyproject.toml` version to `1.2.0` so the fallback path (CI source-tree without pip install, or dev runs without the env var) reports a sensible milestone string.

Purpose: Make DIST-02 partially true (the consumer half — the producer half is Plan 04 `release.yml`). The env-var-first pattern reconciles the project-memory `ci-source-tree-vs-baked-image-divergence` issue without adding `APP_VERSION` to `Settings` (which would break `tests/test_env_example.py` parity).

Output: A Dockerfile that accepts and surfaces `APP_VERSION`, a `system.py` handler that reads it first, and a `pyproject.toml` that no longer says `0.1.0`.
</objective>
