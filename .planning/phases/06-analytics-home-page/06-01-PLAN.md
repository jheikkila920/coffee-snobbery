<objective>
Build `app/services/analytics.py` — "the home page brain" (CLAUDE.md). This is the single query module that every Phase 6 fragment endpoint, the home shell, and Phase 7's stale-data gate consume. It contains nine pure-SQL read functions (eight card derivations + the cold-start gate counts) plus `compute_input_signature`, the Phase 7 plumbing. It also proves the ROADMAP performance budget: every derivation runs <50ms p95 against a 1000-session seed, adding the one justified analytics index only if a query exceeds the budget.

Purpose: Interface-first foundation. Both downstream plans (shell route, fragment endpoints) import from this module, so it must land first with its full public surface defined. Per-user scoping and rating-NULL handling are baked in here so no template or router has to compensate.

Output: `app/services/analytics.py`, `tests/services/test_analytics.py` (Wave-0 seeded-DB tests covering every function), and `tests/services/test_analytics_perf.py` (the 1000-session latency check).
</objective>
