<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → home route | unauthenticated requests must not reach analytics; the shell + every fragment endpoint cross this boundary |
| browser → fragment endpoints | each `/home/cards/*` endpoint is independently reachable; auth must be enforced per-endpoint, not only on the shell |
| query result → template | analytics rows (coffee names, roaster names, descriptors) render into HTML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Information Disclosure | GET / and every /home/cards/* endpoint | mitigate | Every handler declares `Depends(require_user)`; unauthenticated requests get 401 (proven by `test_home_unauthenticated_returns_401` + `test_unrated_coffees_fragment_requires_auth`). |
| T-06-05 | Elevation of Privilege (IDOR) | recent-brews / unrated-coffees | mitigate | Handlers pass `user.id` from `request.state.user`; never read a user_id query param. Service queries are user-scoped (Plan 06-01). |
| T-06-06 | Tampering (XSS) | home.html + fragment templates | mitigate | No `|safe`, no `hx-on:`, no `hx-vals='js:'`; all values autoescaped; CSP-clean probe in Task 2 verify. |
| T-06-07 | DoS (connection pool) | five lazy slots | mitigate | Staggered `hx-trigger="load delay:100..500ms"` spreads fragment requests across 500ms so the `pool_size=10/max_overflow=5` pool is not saturated (HX-5/SH-2). |
| T-06-12 | Reliability (broken cross-phase trigger) | Phase 7 AI-card slot in home.html | mitigate | The Phase 7 AI slot is a Jinja comment, NOT a live `hx-trigger="revealed"` against a non-existent endpoint — prevents a 404-on-reveal in Phase 6 (proven by `test_ai_slot_placeholder_present`). |
</threat_model>
