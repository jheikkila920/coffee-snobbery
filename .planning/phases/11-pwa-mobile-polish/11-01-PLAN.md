<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous client → /manifest.json, /sw.js | Both routes are intentionally public (SW installs before login) |
| service worker → cached responses | SW mediates which network responses are stored and replayed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information disclosure | /manifest.json, /sw.js public routes | accept | Both serve static, non-user-specific content (locked brand strings + cache logic); no auth data, no PII. Confirmed by test_manifest_200 asserting only the locked strings. |
| T-11-02 | Tampering | SW caching a CSRF-protected mutation (POST/PUT/DELETE) | mitigate | sw.js fetch handler first line `if (req.method !== 'GET') return;` — non-GET requests bypass the SW entirely and reach the network with their CSRF token intact (verified by grep in Task 3). |
| T-11-03 | Tampering | SW serving a stale authenticated HTML shell to another user on a shared device | mitigate | APP_SHELL precaches `/` (the analytics home embeds per-user data only via lazy HTMX fragments which are network-first GETs, never cached); the cached `/` shell is the chrome, and per-user fragments always hit the network. Network-first for all non-shell/non-static GETs ensures fresh per-user data. |
| T-11-04 | Tampering | Stale SW shell surviving a deploy (old code served indefinitely) | mitigate | CACHE_NAME embeds __BUILD_HASH__ derived from the Tailwind CSS filename; a `docker compose build` recomputes the CSS hash, changing the cache name; activate handler deletes all non-matching caches (Pitfall 2). |
| T-11-05 | Tampering | `source=pwa` query param injection via start_url | accept | The home route reads no query params and passes none to templates or logs; FastAPI ignores the unknown `source` param. No sink exists. |
</threat_model>
