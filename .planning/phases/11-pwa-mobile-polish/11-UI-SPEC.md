# Phase 11 — UI Design Contract: PWA + Mobile Polish

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Brownfield phase.** The palette, typography, dark-mode tokens, and the dual `theme-color`
> metas are ALREADY SHIPPED. This contract confirms the existing tokens and specifies the
> net-new surfaces only. Do not rebuild what already exists.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | Heroicons (inline SVG — already the established project pattern from base.html + search-bar) |
| Font | System UI stack (Tailwind default — no custom font loaded) |

**Stack constraints (invariants — non-negotiable):**
- No npm build pipeline. No shadcn/ui, Radix, Base UI, or any component library install.
- All CSS in `app/static/css/tailwind.src.css` only. `app/static/css/custom.css` must NOT be created (MX-1 lock).
- All JS is Alpine CSP-build `Alpine.data()` factories registered BEFORE `@alpinejs/csp` core, in `app/static/js/alpine-components/`. No `eval`, no inline `hx-on:`, no inline event handlers.
- `htmx.config.allowEval=false` (enforced in `htmx-listeners.js`).
- CSP is nonce-based; every `<script>` and `<style>` tag must carry `nonce="{{ csp_nonce(request) }}"`.

## Spacing Scale

Source: 8-point Tailwind utility scale (already in use across the codebase). All values are Tailwind spacing tokens; no custom spacing additions needed for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gaps, inline badge padding |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing, nav icon-to-label gap |
| md | 16px (`p-4`, `gap-4`) | Default element spacing, card padding, form section gaps |
| lg | 24px (`p-6`, `gap-6`) | Section padding, sheet header padding |
| xl | 32px (`p-8`, `gap-8`) | GBM timer screen internal padding |
| 2xl | 48px (`py-12`) | Page-level vertical padding (matches existing `py-12` on main) |
| 3xl | 64px | Not needed this phase |

**Exceptions:**
- Bottom tab nav height: `h-16` (64px) — fixed height to provide a stable tap zone above safe-area.
- iOS safe-area: `pb-[env(safe-area-inset-bottom)]` appended to bottom nav padding. Use `viewport-fit=cover` (already set in `base.html`).
- GBM countdown timer text: `text-7xl` or `text-8xl` — deliberately oversized, not a spacing value.
- Tap target minimum: 44×44px enforced with `min-h-[44px] min-w-[44px]` (already the project pattern from search-bar buttons). Apply to ALL interactive controls: nav tabs, GBM controls, close buttons, sheet triggers.
- Sticky form actions (BREW-08) bottom offset: `bottom-16` (64px = nav height) + `pb-[env(safe-area-inset-bottom)]` on mobile so Save/Cancel sit above the persistent nav.

## Typography

Source: Extracted from `tailwind.src.css` `@layer base`, `base.html`, and `tailwind.config.js`. These values are already shipped — confirming, not changing.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base` |
| Label / small UI | 14px | 400 (regular) | 1.4 | `text-sm` |
| Heading (section) | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Display (page h1 / GBM timer) | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |

**GBM timer exception:** The countdown value renders at `text-7xl font-semibold` (72px). This is intentional — the feature's primary affordance is legibility at arm's length. It is NOT a general typography token.

**Input floor:** 16px minimum on all `input`, `select`, `textarea` — already enforced in `tailwind.src.css @layer base` (MX-1, MOB-06). Do not override below 16px anywhere.

**Nav tab labels:** `text-xs` (12px) below the icon, weight 400, line-height 1. Acceptable for nav chrome at standard device scaling; does NOT violate the 16px input-floor rule (that rule applies to form controls only).

#### 3b. Service Worker (`/sw.js`) — MOB-10

Strategy (locked by ROADMAP success criterion #2):
- **App shell (stale-while-revalidate):** `base.html` output, `tailwind.<hash>.css`, Alpine JS, HTMX JS, manifest, icons.
- **Non-GET requests:** bypass (no caching).
- **All other GET requests:** network-first.
- **Cache name:** embeds build hash (e.g. `snobbery-v{BUILD_HASH}`) so each deploy purges old shells.
- **Header on `/sw.js` response:** `Service-Worker-Allowed: /` (PWA-3).
- **NGINX cache directive (README/docs):** `Cache-Control: no-cache` on `/sw.js` (PWA-7).

Offline writes show a clear error (`"You're offline. Changes cannot be saved right now."`) — no background sync in v1.

This is the largest net-new surface. No existing analog. Highest design-detail priority.

When the last step completes:
1. Show completion screen: "Brew complete! Time: MM:SS" (`text-2xl font-semibold`).
2. Two CTAs:
   - "Log this brew" (`bg-espresso-700 text-cream-50`) — navigates to `/brew/new?gbm=1&recipe_id=N[&coffee_id=M]&brew_time=T` with prefill.
   - "Done without logging" (`text-sm text-espresso-600 underline`) — navigates home.

#### 4e. Wake Lock + iOS Fallback (BREW-13, D-11)

| State | Indicator |
|-------|-----------|
| Wake lock held | "● Screen stays on" — small pill in top-right corner of timer screen. Green dot (`text-green-600`) + `text-xs text-espresso-600 dark:text-cream-300`. |
| Wake lock not held (iOS fallback active) | Same indicator, yellow dot (`text-yellow-500`): "● Screen kept on (fallback)" |
| Wake lock failed (no fallback) | No indicator; no error shown — silent degradation. |

**Implementation notes for executor:**
- Request `wakeLock.request('screen')` on GBM start.
- Re-acquire on `document.addEventListener('visibilitychange', ...)` when `document.visibilityState === 'visible'` (MX-4).
- iOS fallback (prototype on real iPhone before declaring done — carried research flag): silent audio loop OR NoSleep.js. Either must load as a module registered in the CSP (nonce-tagged script or bundled into an Alpine component). Decision deferred to plan-phase after prototype.
- All wake-lock JS lives in `app/static/js/alpine-components/guided-brew-mode.js` (`Alpine.data('guidedBrewMode', ...)`).

#### 4f. Audio Chime + Vibration (D-09)

- **Chime:** `AudioContext` + short synthesized tone (no external audio file dependency). iOS requires a user gesture to unlock AudioContext — the "Start guided brew" button serves as the gesture unlock.
- **Vibration:** `navigator.vibrate([100, 50, 100])` (two short pulses). Fails silently on iOS (vibrate API unsupported); no user-visible error.
- Both cues gated by the localStorage toggle booleans (`snobbery:gbm:cues` → `{ chime: true, vibrate: true }`).

#### 4g. Cancel Without Logging Confirmation

Pattern: `window.confirm('Cancel brew? This cannot be undone.')` — same pattern as the existing mini-modal dirty-check in `mini-modal.js`. No custom modal needed.
