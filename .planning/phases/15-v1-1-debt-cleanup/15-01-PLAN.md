<objective>
Fix G-01 (DEBT-01): the runtime container can no longer write backups/photos to a
pre-existing root-owned `/app/data` named volume without a manual `chown`. Rewrite the
privilege model so the container starts as root, conditionally chowns `/app/data`, then
drops to the unprivileged `app` user (UID 1000) via `gosu` before `exec`-ing uvicorn.

Purpose: A new operator's first deploy must "just work" — no out-of-band `docker compose
run --rm -u root ... chown` step. This is the gate for Phase 18 (Self-Host Packaging) and
removes the carried Phase 08 deferral G-01.
Output: Rewritten `entrypoint.sh` (root -> conditional chown -> gosu drop) and a two-line
`Dockerfile` runtime-stage change (add `gosu` to the existing apt block; remove `USER app`).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host -> container init | The container's PID 1 starts as root (required to chown the named volume), then permanently drops to UID 1000 before any request is served |
| named volume -> container fs | A host-provisioned `/app/data` volume may be root-owned on first deploy; the entrypoint normalizes ownership at boot |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Elevation of Privilege | container runtime process (entrypoint -> uvicorn) | mitigate | `exec gosu app uvicorn` drops to UID 1000 before uvicorn binds; no request is ever served as root. The root window is the entrypoint's chown+migrate only. Acceptance criteria assert the running uvicorn process is UID 1000. |
| T-15-02 | Elevation of Privilege | `gosu` setuid binary added to image | accept | gosu is invoked ONLY by the entrypoint at startup, never by a request handler; this is the canonical pattern used by the official Postgres image. No new capability is granted to the container (no `--privileged`, no added cap). Risk accepted per RESEARCH Security Domain. |
| T-15-03 | Denial of Service | `docker stop` shutdown path | mitigate | The `exec` before `gosu` makes uvicorn PID 1, so SIGTERM forwards directly for a clean <3s shutdown rather than a 10s SIGKILL fallback that can drop in-flight DB work. Task 3 asserts the stop time. |

No new capabilities are granted; the container runs unprivileged (UID 1000) after startup. No auth, session, crypto, or input-validation surface changes in this plan.
</threat_model>
