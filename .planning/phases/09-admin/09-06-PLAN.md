<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| handler -> app_settings status rows | Cache-pop gotcha; must read raw or the panel crashes after a write |
| admin form -> ai_service.regenerate | Force mode re-bills every user; must be deliberate + tagged; never add a hidden cost ceiling |
| handler -> ai_recommendations error display | Stored error_status may contain untrusted provider text; must be escaped + truncated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-25 | Denial of Service | get_str raising on status rows after a write | mitigate | Raw select(AppSetting...) for last_ai_run_status + last_backup_status (Pitfall 2); test_health_panel_raw_db proves no get_str |
| T-09-26 | Information Disclosure | stored error_status rendered as raw HTML (XSS) | mitigate | Jinja autoescape ON; truncate; never |safe; test_health_panel_errors asserts escaped output |
| T-09-27 | Spoofing / cost abuse | accidental force-refresh re-bills all users | mitigate | Force button explicitly labeled; distinct generated_by="admin_force" for telemetry; respect-signature is the default; no new ceiling per PROJECT |
| T-09-28 | Denial of Service | async regenerate / blocking the loop | mitigate | regenerate is awaited (async def handler) and brackets its own sync DB I/O; sequential iteration; reuse of single-worker-safe locks inside regenerate |
| T-09-29 | Elevation of Privilege | non-admin reaching system/health/refresh | mitigate | require_admin on GET /system + POST /system/ai-refresh; non-admin 403 |
</threat_model>
