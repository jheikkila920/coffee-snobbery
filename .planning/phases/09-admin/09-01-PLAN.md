<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /admin/* | Any authenticated user can attempt admin routes; only is_admin may pass |
| browser -> home.html | The admin link is rendered only for is_admin; a non-admin must not see or reach /admin |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Elevation of Privilege | every /admin route | mitigate | Depends(require_admin) on the package router / every handler; test_non_admin_403 covers the hub route |
| T-09-02 | Information Disclosure | home.html admin link | mitigate | is_admin Jinja guard; non-admin never sees the link (defense-in-depth; the 403 gate is the real control) |
| T-09-03 | Spoofing | admin section nav links | accept | Links are plain anchors to gated routes; no state change; the gate enforces access |
</threat_model>
