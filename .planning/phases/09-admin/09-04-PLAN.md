<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin form -> settings service | Arbitrary key/value submission; only non-read-only keys may be written; coercion is server-side |
| handler -> app_settings status rows | Must read raw (cache pop gotcha); must never let setup_completed flip silently |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-15 | Tampering | editing setup_completed re-opens /setup | mitigate | _READ_ONLY_KEYS guard rejects the save with 403 before set_setting; test_readonly_rows_rejected |
| T-09-16 | Tampering | arbitrary value bypassing type coercion | mitigate | set_setting is the single coercion point (value_type-aware); router passes the raw string, never writes directly |
| T-09-17 | Denial of Service | get_str raising on status rows after a write | mitigate | Raw select(AppSetting...) for status rows (Pitfall 2); never get_str |
| T-09-18 | Tampering | CSRF on inline save | mitigate | Hidden X-CSRF-Token field per row form; CSRFFormFieldShim validates |
| T-09-19 | Elevation of Privilege | non-admin editing settings | mitigate | require_admin on list + save; non-admin 403 |
</threat_model>
