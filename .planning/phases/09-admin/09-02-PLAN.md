<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> /admin/users/* | Admin-only; user IDs come from the URL path, never trusted form body |
| handler -> sessions table | Privilege/active changes must evict stale cookies immediately |
| handler -> users table | Destructive deletes must never silently cascade brew history |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Elevation of Privilege | session not invalidated after is_admin/deactivate | mitigate | Async bulk delete(SessionModel).where(user_id=target) on toggle + deactivate + delete; test_toggle_admin_invalidates_sessions asserts count==0 |
| T-09-05 | Denial of Service | last-admin lockout / self-lockout | mitigate | SELECT COUNT(*) ... FOR UPDATE guard before any demote/deactivate/delete (Pitfall 7); self-id check; tests cover both |
| T-09-06 | Tampering | hard-delete cascades brew history (data loss) | mitigate | D-15: brew_sessions count guard returns 409 before DELETE; RESTRICT FK is the DB backstop; test_delete_user_with_sessions_blocked |
| T-09-07 | Elevation of Privilege | IDOR on target user_id | mitigate | user_id from path param + require_admin gate; never from form body |
| T-09-08 | Information Disclosure | password hash / plaintext in template or log | mitigate | Edit form never renders password_hash; structlog emits event + ids only, never password |
| T-09-09 | Tampering | CSRF on user mutations | mitigate | CSRFFormFieldShim + hidden X-CSRF-Token field in every form |
</threat_model>
