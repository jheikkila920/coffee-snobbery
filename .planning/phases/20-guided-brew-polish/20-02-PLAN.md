| stored JSONB steps → StepSchema | Old step dicts deserialized through the extended schema |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-02 | Spoofing | POST /water-profiles | mitigate | Depends(require_user) on every endpoint; household-shared so no per-user ownership gate (ASVS V4, RESEARCH Security Domain) |
| T-20-03 | Tampering | CSRF on POST /water-profiles | mitigate | starlette-csrf middleware covers the new POST (not exempt); router strips X-CSRF-Token from form before validation |
| T-20-04 | Tampering | Mass assignment on WaterProfileCreate | mitigate | ConfigDict(extra="forbid") rejects unknown keys; router reads raw form then validates via Pydantic |
| T-20-05 | Tampering | water_profile_id integer tampering | mitigate | Field(None, ge=1) in BrewSessionCreate + FK constraint; ondelete SET NULL nulls invalid refs |
| T-20-06 | Tampering | Migration SQL injection via stored water_type | accept | water_type values are parameterless within INITCAP/TRIM set operations; no string concat of user data into DDL; values already in DB |
| T-20-07 | DoS | StepSchema unbounded note/temp | mitigate | note max_length=200, water_temp_c bounded 50-100, water_grams le=2000 (ASVS V5) |
