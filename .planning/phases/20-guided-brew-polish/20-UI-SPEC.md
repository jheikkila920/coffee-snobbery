## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`) | Icon gaps, inline chip spacing |
| sm | 8px (`gap-2`, `px-2`, `py-2`) | Compact element spacing, step-builder field gaps |
| md | 16px (`px-4`, `py-4`, `gap-4`) | Default element spacing, card internal padding |
| lg | 24px (`px-6`, `py-6`, `gap-6`) | Page horizontal padding, section gaps |
| xl | 32px (`gap-8`) | Layout-level gaps |
| 2xl | 48px | Major section breaks (rare; use sparingly in Guided Brew) |
| 3xl | 64px | Display number height (coach countdown, `text-7xl`) |

Exceptions:
- `py-3` (12px) is used on secondary button rows — this pre-existing pattern is preserved.
- Pre-cue countdown display uses `text-7xl` (72px display height) — matches existing countdown size on the timer screen.
- Per-step note field uses `py-2 px-2` (8px) — compact, matching existing step-builder inputs.

## Typography

Source: existing codebase patterns confirmed in `brew_guided.html` and `recipe_step_builder.html`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 72px (`text-7xl`) | 600 (`font-semibold`) | 1.0 (`leading-none`) | Coach countdown, step remaining time |
| Heading | 24px (`text-2xl`) | 600 (`font-semibold`) | 1.2 | Screen titles ("Start guided brew", "Brew complete!") |
| Body | 16px (`text-base`) | 400 (`font-normal`) | 1.5 | Step labels, coaching line, general content |
| Label | 14px (`text-sm`) | 600 (`font-semibold`) or 400 | 1.4 | Form field labels (`font-semibold`), metadata, step counter, secondary info |

Two weights only: 400 (regular) and 600 (semibold). Do not introduce 500 or 700.

Numeric data (timers, water targets, grams): always `tabular-nums` class for alignment stability.

Form inputs: 16px minimum enforced globally via `@layer base` in `tailwind.src.css` — do not use `text-sm` on inputs (triggers iOS zoom).

## Color Contract (60/30/10)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `cream-50` / `espresso-950` | Full-screen Guided Brew container bg; page surfaces |
| Secondary (30%) | `cream-100` / `espresso-900` | Step preview list, bordered cards, form input backgrounds, nav |
| Accent (10%) | `espresso-700` / `cream-50` | Reserved for specific elements below |
| Destructive | `red-700` (Tailwind default) | "Cancel without logging" only — text link, no background fill |

**Accent (`espresso-700`) is reserved for exactly these elements:**
1. Primary CTA buttons ("Start guided brew", "Log this brew", "Save water profile")
2. Currently active step card (full `bg-espresso-700 text-cream-50` card)
3. Active/on state of toggle buttons (chime, vibrate, On/Off pairs)
4. Progress bar fill
5. Coach view: current step type badge background
6. Tap-to-mark buttons (first-drip, bloom) in their "mark it" active state
7. Water-profile inline save trigger

Accent is NOT used on: metadata text, step counters, secondary descriptive copy, next-step preview, navigation items in non-active state.

Dark mode: `espresso-700` surfaces flip to `cream-50` text (existing pattern). Dark-mode body background is `espresso-950`. All new UI follows the `.dark` selector pattern — never `@media (prefers-color-scheme: dark)`.

## New UI Surfaces — Interaction Contracts

### 1. Water Profile Inline Select-or-Create (GBREW-04)

Pattern model: `observedFlavorNotes` / `flavor-tag-input.js`.

**Select control:** Native `<select>` listing all existing water profiles by name, with a trailing "Add new..." option that toggles to create mode. Not a tag chip (water profile is a single scalar FK, not multi-select). Alpine.js inline — no separate file needed unless >80 LOC.

**Create mode:** On "Add new..." selection, the `<select>` is replaced (via `x-show`) with:
- Text input: placeholder "Profile name (e.g. Third Wave Water)", `maxlength="80"`, `text-base`, `rounded border border-espresso-200 px-2 py-2`
- Optional notes input: `<textarea rows="2">`, placeholder "Optional notes", same styling
- Save button: `min-h-[44px] rounded bg-espresso-700 text-cream-50 px-4 text-sm font-semibold`
- Cancel link: `text-sm text-espresso-600 underline min-h-[44px] inline-flex items-center`

**HTMX interaction:** Save POSTs to `/water-profiles` with CSRF token (via global `htmx:configRequest` listener). On `HX-Trigger: water-profile-created`, the Alpine component adds the new profile to the select and selects it. Swap: `none` (listener-driven, mirrors flavor-note pattern).

**Form field:** `name="water_profile_id"` (integer FK). Replaces the existing `name="water_type"` freetext input.

**Validation state:** If profile name is empty on save attempt, show `text-sm text-red-600` below the input: "Profile name is required."

**Empty catalog state (first-time user):** Select shows only "Add new..." as the option. No separate empty state heading needed — the placeholder copy handles it.

### 2. Recipe Step Builder Extensions (GBREW-06, D-04..D-07)

Extends `app/templates/fragments/recipe_step_builder.html` with three new fields per step row.

**Step Type selector (D-04):**
- `<select name="step_type" ...>` with options: Bloom / Pour / Wait / Action
- Inline with the existing Label field: `flex flex-wrap gap-3 items-end`
- Width: `w-32` — narrow, sits left of Label
- Label text: `<span class="text-sm font-semibold">Type</span>`
- Default on `addStep()`: "Pour"

**Per-step note field (D-05):**
- `<textarea rows="2" maxlength="200">` below the existing label/water/time row
- Placeholder: "Optional step note (e.g. Hario Switch closed for immersion)"
- Full width: `w-full`
- Styling: `rounded border border-espresso-200 px-2 py-2 text-base`
- Only shown when user has started typing OR when step already has a note — controlled via `x-show="step.note || showNote[idx]"` with a small "Add note" text trigger: `text-sm text-espresso-600 underline cursor-pointer`

**Per-step water temperature (D-06):**
- `<input type="number" inputmode="decimal">` in the existing flex row alongside Water (g) and Time (s)
- Width: `w-24` — matches Water and Time fields
- Label: `<span class="text-sm font-semibold">Temp (°C)</span>`
- Placeholder: "—" (optional, most steps leave blank)
- Range: `min="50" max="100"`

**Wait/Action step UI behavior (D-07):**
- When step type is Wait or Action, the Water (g) field is visually dimmed (`opacity-40`) and its value cleared (not required)
- The step row's border color changes to `border-espresso-300 dark:border-espresso-600` (lighter) to signal non-pour step type
- Coaching label auto-generates: "Wait" → "Wait — {time}", "Action" → use Label text directly

**Step row dimensions:** Existing `w-11 h-11` reorder and remove buttons retained. No new touch target sizes introduced.

### 3. Tap-to-Mark Buttons (GBREW-03, D-12..D-14)

Shown on the Guided Brew **Timer screen** during live brew. Both are inline on the coach view.

**First-drip button:**
- Label: "Mark first drip"
- State after tap: shows elapsed time since brew start (e.g. "First drip: 0:23") with a small "✓" indicator and option to "Clear" (text link)
- Size: `min-h-[56px] w-full rounded bg-espresso-700 text-cream-50 text-base font-semibold flex items-center justify-center`
- After mark: button becomes a display tile: `rounded bg-espresso-50 dark:bg-espresso-900 border border-espresso-200 dark:border-espresso-700 px-4 py-3 text-sm flex items-center justify-between`

**Bloom time capture (D-13 — auto-derives from Bloom step):**
- Not a tap button during guided brew — bloom time auto-records when the Bloom-type step's countdown completes
- Display-only indicator on the coach view during Bloom step: "Bloom will auto-record"
- After Bloom step ends: shows "Bloom: {m}:{ss}" as a small `text-sm text-espresso-600 dark:text-cream-300` line

**Manual entry on session form (D-12, non-guided sessions):**
- Two new optional number inputs on `brew_form.html`: "First drip (s)" and "Bloom time (s)"
- Same styling as existing time inputs: `w-24 rounded border border-espresso-200 px-2 py-3 text-base tabular-nums`
- Grouped under a `<fieldset>` labeled "Brew timing (optional)"

### 4. Full Coach View — Timer Screen Redesign (GBREW-02, D-11)

Replaces / extends the existing timer screen layout. All content within the existing `x-show="isRunning && !isDone"` block.

**Layout hierarchy (top to bottom, mobile-first):**

```
[ Wake lock indicator — text-xs, centered ]
[ Step counter — "Step 2 of 4", text-sm, text-espresso-600 ]
[ PRE-CUE COUNTDOWN — shown 3s before transition only ]
  text-2xl font-semibold text-espresso-600 "Get ready…"
  + 3-2-1 number: text-5xl font-semibold tabular-nums espresso-700
[ CURRENT STEP CARD — full bg-espresso-700 text-cream-50 rounded-lg px-4 py-5 ]
  Step type badge: text-xs font-semibold uppercase tracking-wide, cream-50/70 opacity
  Coaching line: text-xl font-semibold (e.g. "Pour 2 — to 250g")
  Per-step note: text-sm opacity-80 (e.g. "Hario Switch closed") — hidden if empty
  Step water temp: text-sm opacity-70 "at 94°C" — hidden if no per-step temp
[ REMAINING COUNTDOWN — text-7xl font-semibold tabular-nums, centered ]
[ PROGRESS BAR — existing: w-full h-1 bg-espresso-200, fill espresso-700 ]
[ CUMULATIVE WATER TARGET — "180g / 300g total", text-base tabular-nums espresso-700 font-semibold ]
[ TOTAL ELAPSED — "Elapsed: 1:23", text-sm tabular-nums espresso-600 ]
[ NEXT STEP PREVIEW — "Next: Wait — 30s", text-sm espresso-600 ]
[ FIRST-DRIP BUTTON (or marked display) ]
[ PAUSE / NEXT STEP buttons — existing layout, min-h-[56px] ]
[ CUE TOGGLES — existing, min-h-[44px] ]
[ Cancel without logging — bottom, mt-auto, min-h-[44px] ]
```

**Pre-cue countdown (D-10):**
- Triggers 3 seconds before step transition
- Replaces the main countdown display with "Get ready…" + 3-2-1 counter
- Color: `text-espresso-600 dark:text-cream-300` for "Get ready…"; `text-espresso-700 dark:text-cream-100 text-5xl font-semibold tabular-nums` for the digit
- The step card stays visible beneath it
- Audio + vibration cue fires at the transition (not at pre-cue start) — D-08 locked

**Coaching line auto-composition (D-09):**
- Bloom step: "Bloom — {water_grams}g" (if water_grams present) or "Bloom"
- Pour step: "Pour {n} — to {cumulative_water}g" where n is pour sequence number (1-indexed among Pour steps)
- Wait step: "Wait — {time_formatted}" (no water reference)
- Action step: use `step.label` directly (e.g. "Open Hario Switch")

**Step type badge values:** `BLOOM` / `POUR` / `WAIT` / `ACTION` — uppercase, `text-xs font-semibold tracking-wide`

## Accessibility Contract

- All new buttons must have visible label text (no icon-only buttons without `aria-label`)
- Tap-to-mark buttons: `aria-label="Mark first drip time"` / `aria-label="Clear first drip time"`
- Step type select: `aria-label="Step type for step {n}"`
- Pre-cue countdown: `aria-live="polite"` on the countdown container so screen readers announce the number change
- Coach view current step card: `aria-live="assertive"` — step transitions are high-signal events
- Water profile select: standard `<label>` wrapping with `for` attribute
- Form inputs: 16px minimum (globally enforced); no exceptions

## CSP + Dark Mode Invariants

Source: CONTEXT.md `<code_context>` + CLAUDE.md architectural invariants.

- No `|safe` filter on any user-supplied data in new templates
- No inline `<script>` without `nonce="{{ csp_nonce(request) }}"` — all new JS loaded via `<script defer src=... nonce=...>`
- No `eval(`, `new Function`, or dynamic `import()` in new Alpine component code
- CSRF token on every state-changing form and HTMX POST — new water-profile create endpoint is not exempt
- Dark mode: all new Tailwind classes must include `dark:` variants using `.dark` selector convention — never write `dark:` variants that only work under `prefers-color-scheme`. Pattern: `bg-cream-50 dark:bg-espresso-950`, `text-espresso-700 dark:text-cream-200`, `border-espresso-200 dark:border-espresso-700`
- `tailwind.src.css` custom classes (`.nav-safe-area`, `.nav-safe-area-extend::after`, `.content-nav-safe-area`) must be applied to any new fixed-position or scrollable content areas introduced in Guided Brew — specifically the full-screen coach view container already uses `fixed inset-0 z-50`; the existing classes apply

## Safe-Area Application

Source: CONTEXT.md D-16 + commit `982c0e6` (verified on John's physical iPhone PWA).

The Guided Brew container uses `fixed inset-0 z-50` — it covers the bottom nav. Safe-area rules:
- Apply `pb-[env(safe-area-inset-bottom)]` to the cancel button / bottom action area inside the coach view container (the coach view is its own scroll container, not the document body)
- The coach view container itself does NOT use `.content-nav-safe-area` (that is for page-body content beneath the fixed nav)
- For the tap-to-mark button area and the cancel link, add explicit bottom padding: `pb-[max(env(safe-area-inset-bottom),_16px)]` to the `mt-auto` bottom section
