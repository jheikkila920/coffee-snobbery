<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uploaded CSV bytes -> importer | Untrusted file: content-type + size guard before buffering; decode defensively; never eval row data |
| import row fields -> SQL resolution | Name matches must be parameterized; user_id never taken from the file |
| export cells -> downstream spreadsheet | Exported free-text could carry a formula payload when opened in Excel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | Tampering (SQLi) | coffee/bag/flavor-note name resolution | mitigate | SQLAlchemy `select()` + bound params; citext match via the ORM, never string-formatted SQL (Task 1) |
| T-05-10 | Elevation (mass assignment) | import row setting user_id / extraction_yield_pct | mitigate | user_id from by_user_id only; EY never written (GENERATED); brew_csv schema extra=forbid (Task 1) |
| T-05-11 | DoS (oversized upload) | import upload | mitigate | Reject non-CSV content-type + size ceiling before buffering full body (Task 1) |
| T-05-12 | Integrity | partial commit on mid-file error | mitigate | Resolve+validate all rows, then single-transaction insert + one commit; rollback on error (Task 1, Pitfall 4) |
| T-05-13 | Tampering (downstream) | CSV formula injection in export | mitigate | Prefix leading `= + - @` with `'` on free-text export columns (Task 2); recorded as the chosen mitigation |
| T-05-14 | Info disclosure | exporting another user's sessions | mitigate | Export query scoped by by_user_id (Task 2) |
</threat_model>
