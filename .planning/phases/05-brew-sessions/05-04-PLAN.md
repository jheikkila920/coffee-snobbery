<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client form/POST -> brew router | Untrusted form; Pydantic validates; user_id + EY never taken from input; CSRF on every POST |
| URL path /brew/{id} -> service | session_id from the URL must be checked for ownership before any read/write |
| draft autosave POST -> brew_drafts | Must carry CSRF; keyed by server user_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-15 | Elevation / Info disclosure (IDOR) | GET/POST /brew/{id}, /brew/new?from= | mitigate | get/update scoped by user.id; 404 (not 403) on non-owned id; from= source is user-scoped (Task 1) |
| T-05-16 | Tampering / Elevation (mass assignment) | POST /brew payload | mitigate | _parse_form_payload never reads user_id/extraction_yield_pct; BrewSessionCreate extra=forbid folds extras to _form; server sets user_id from request.state.user.id (Task 1) |
| T-05-17 | Spoofing (CSRF) | POST /brew, POST /brew/{id}, POST /brew/draft | mitigate | Hidden X-CSRF-Token field + global htmx-listeners.js header; no route exempted (Tasks 1,2) |
| T-05-18 | Info disclosure | server draft cross-user read | mitigate | get_draft keyed by user.id; never returns another user's draft (Task 2) |
</threat_model>
