  <name>Task 0: Confirm EY formula unit + user_id ondelete</name>
  <decision>Two schema specifics must be locked before the migration writes the GENERATED clause: (1) the exact extraction-yield expression (depends on whether tds_pct is stored as a whole percent or a fraction), and (2) the user_id FK ondelete policy.</decision>
  <context>extraction_yield_pct is a Postgres GENERATED column — a wrong unit produces EY values off by 100x for every refractometer brew, and changing a GENERATED expression later is a migration. The user_id ondelete affects how admin user-delete (Phase 9 ADMIN-01) treats a user's logs. CONTEXT and RESEARCH flag both (A4/Q2, Pitfall 5/Q3); the original spec doc is absent from disk to settle it.</context>
  <options>
    <option id="tds-whole-percent">
      <name>tds_pct stored as whole percent (e.g. 1.35 = 1.35%)</name>
      <pros>Matches how a refractometer reads out and the "TDS %" UI label; intuitive to type at the kettle</pros>
      <cons>GENERATED expression must divide by 100: (yield_grams_actual * tds_pct / 100.0) / dose_grams_actual</cons>
    </option>
    <option id="tds-fraction">
      <name>tds_pct stored as fraction (e.g. 0.0135)</name>
      <pros>CONTEXT's literal formula works as written, no /100</pros>
      <cons>Diverges from the "TDS %" UI label; users would type 0.0135 not 1.35; error-prone</cons>
    </option>
    <option id="userdel-restrict">
      <name>user_id ondelete=RESTRICT (recommended)</name>
      <pros>Deleting a user who has logs is a conscious admin action; brew history never silently vanishes</pros>
      <cons>Phase 9 admin user-delete must explicitly handle logs first</cons>
    </option>
    <option id="userdel-cascade">
      <name>user_id ondelete=CASCADE</name>
      <pros>Deleting a user cleans up their logs automatically</pros>
      <cons>Irreversible loss of brew history + analytics on a single admin action</cons>
    </option>
  </options>
  <resume-signal>State: (a) tds-whole-percent or tds-fraction, and (b) userdel-restrict or userdel-cascade. Recommended default with no preference: tds-whole-percent + userdel-restrict.</resume-signal>
