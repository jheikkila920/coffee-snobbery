<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| router -> service (session_id, user_id) | Service must scope every read/write/delete by user_id; a session_id alone must never grant access |
| draft payload (client JSON) -> brew_drafts | Stored opaque; never eval'd; keyed by server-derived user_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Elevation / Info disclosure (IDOR) | get/update/delete_brew_session; resolve_prefill from_session_id; get/clear_draft | mitigate | Every query filtered by user_id; update/get/delete return None/False (router maps to 404) for non-owned ids; draft keyed by server user_id (Tasks 1,2,3) |
| T-05-06 | Tampering (SQLi) | list filters, citext/name matches, prefill queries | mitigate | SQLAlchemy `select()` parameterized only; no string-formatted SQL (Tasks 1,2) |
| T-05-07 | Integrity | usage_count drift on edit/delete | mitigate | Same-transaction diff across all three FKs; -1 on delete (Task 1, Pitfall 6) |
| T-05-08 | Info disclosure | draft cross-user leak (server side) | mitigate | brew_drafts UNIQUE user_id; all ops keyed by by_user_id; clear on submit+logout (Task 3, MX-5) |
</threat_model>
