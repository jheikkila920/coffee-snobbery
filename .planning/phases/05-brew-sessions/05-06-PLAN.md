<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filter query params -> SQL | Untrusted filters parameterized in list/export queries |
| uploaded CSV -> import route | content-type + size guard; CSRF; user_id server-set |
| list/export -> response | Only the authed user's sessions may ever be returned |
| refused-reason strings -> DOM | Autoescaped (no |safe) — stored-XSS defense |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-24 | Info disclosure / Elevation (IDOR) | GET /brew, GET /brew/export | mitigate | list/export queries scoped by user.id; a second user's sessions never returned (Task 1, test_list_user_scoped) |
| T-05-25 | Tampering (SQLi) | filter params in list/export | mitigate | SQLAlchemy `select()` parameterized; filters bound, never string-formatted (Task 1) |
| T-05-26 | Spoofing (CSRF) | POST /brew/import | mitigate | CSRF enforced; route not exempt; hidden token field on the import form (Tasks 1,2) |
| T-05-27 | DoS (oversized upload) | POST /brew/import | mitigate | content-type + size ceiling before buffering (Task 1, csv_io guard) |
| T-05-28 | Tampering (stored XSS) | csv_import_results.html refused reasons + session_row names | mitigate | Jinja autoescape; no `|safe` (existing grep test covers new templates) (Task 2) |
| T-05-29 | Tampering (downstream) | exported CSV opened in Excel | mitigate | csv_io export prefixes leading `= + - @` (Plan 03 Task 2) — inherited here |
</threat_model>
