<objective>
Stop showing raw `{"detail":"Authentication required"}` JSON to logged-out browsers.

A request to any `require_user`-gated page (e.g. `/`) currently raises an uncaught
`HTTPException(401)`, so FastAPI renders its default JSON body. A browser at the site
root sees JSON instead of the login page.

Fix: add ONE global exception handler in `app/main.py`, registered on the Starlette
base `HTTPException` (FastAPI's `HTTPException` subclasses it, so one registration
catches both). The handler branches ONLY on `status_code == 401`:
- full-page browser nav -> 303 redirect to `/login`
- HTMX request -> 401 with `HX-Redirect: /login` (client-side redirect, no DOM swap)
- everything else -> default FastAPI behaviour

All non-401 statuses delegate to FastAPI's default handler unchanged.

Purpose: a logged-out user who opens the app lands on the login form, not a JSON blob,
without weakening the deliberate JSON-403 admin no-leak behaviour or any security middleware.
Output: the handler + its registration in `app/main.py`, and a focused test module.
</objective>
