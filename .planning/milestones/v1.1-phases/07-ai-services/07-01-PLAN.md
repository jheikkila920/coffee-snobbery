<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AI provider response → app | LLM/tool output (incl. web-search results) is untrusted input crossing into structured parsing |
| AI-suggested buy_url → outbound fetch | An AI-generated URL is an attacker-influencable SSRF vector when fetched for verification |
| browser → AI POST routes | Untrusted client requests (handled in 07-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure / Tampering (SSRF) | `_verify_buy_url` ranged GET | mitigate | https-only scheme allowlist before any request; `follow_redirects=False` (no cross-host redirect to 169.254.169.254 / internal ranges); 64KB Range cap; 5s timeout. Tests: test_url_verify_scheme_rejected, test_url_verify_ssrf_redirect (HIGH). |
| T-07-02 | Tampering / Elevation (prompt injection) | `_project_tool_use_input` + Pydantic schemas | mitigate | Keep-only-named-tool_use projection discards web-search text/result blocks; `ConfigDict(extra="forbid")` rejects injected fields; ValidationError → "try_again" (07-03 consumes). Test: test_citation_projector. |
| T-07-03 | Information Disclosure (secret leakage) | `_build_*_client`, `_write_recommendation_row`, structlog | mitigate | cred.key passed only to SDK constructor; never logged, never placed in a Pydantic model or response_json; only cred.provider/model_name logged (SEC-6). |
| T-07-04 | Denial of Service (unbounded state growth) | `_THROTTLE` module dict | mitigate | `_evict_stale_throttle` removes entries older than 10-min window on access (Pitfall 8). |
</threat_model>
