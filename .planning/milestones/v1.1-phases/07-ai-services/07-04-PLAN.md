<objective>
Implement the two on-demand flows that are NOT part of the cached nightly bundle: the equipment
recommendation (D-05 — profile-only, no web search, generate-on-click) and paste-and-rank (D-07/D-08 —
dedicated flow accepting both pasted text and URLs, never cached, never scheduled). The paste-rank URL
path reuses the AI-05 ranged-GET safety machinery (realistic UA, 5s timeout, no cross-host redirects)
for bounded page-text extraction, which is SEPARATE from the live-rec web search.

Purpose: These flows extend the differentiator surface. Equipment rec answers "what's my weakest link";
paste-rank answers "rank these N coffees against my taste." Both are user-triggered and write a telemetry
row but never participate in signature-based regeneration.

Output: equipment + paste-rank flow logic appended to `app/services/ai_service.py` + tests.

DISCRETION RESOLVED: paste-rank URL extraction depth = bounded to ~128KB ranged GET, extract <p>/<h1>/<h2>
text via stdlib html.parser, cap the model-fed text at ~8000 tokens of extracted content (RESEARCH Pattern 6
paste-rank note). equipment rec + paste-rank both write rows (rec_type "equipment" / "paste_rank") for telemetry
but are excluded from regenerate()/signature logic.
</objective>
