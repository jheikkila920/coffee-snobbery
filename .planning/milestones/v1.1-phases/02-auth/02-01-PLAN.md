Fixture 2 — `fresh_db` (AUTOUSE=True for Phase-2 setup-flow tests; required by AUTH-01 / AUTH-02):

```
@pytest.fixture(autouse=True)
def fresh_db() -> Iterator[None]:
    """Reset users + app_settings.setup_completed before each test.

    The AUTH-01 / AUTH-02 tests REQUIRE setup_completed='false' and zero users.
    """
    try:
        from app.db import engine
    except ImportError:
        yield
        return
    try:
        from sqlalchemy import text
    except ImportError:
        yield
        return
    try:
        with engine.begin() as conn:
            # Cascades to sessions (FK with ON DELETE CASCADE per session model)
            conn.execute(text("TRUNCATE TABLE users RESTART IDENTITY CASCADE"))
            conn.execute(text("DELETE FROM sessions"))
            conn.execute(text(
                "UPDATE app_settings SET value='false' WHERE key='setup_completed'"
            ))
    except Exception:
        # Postgres unreachable — let the test that needs DB skip itself via its own probe
        pass
    yield
```

The `fresh_db` fixture is autouse so EVERY Phase-2 test starts with a clean slate; tests that don't touch the DB pay only the TRUNCATE cost (~1 ms). The try/except wrapping the DB connection means unit-only test runs (no Postgres) don't error — the test that genuinely needs the DB will skip via its own `try/except ImportError` probe.

Fixture 3 — `_seed_user(is_admin: bool)` helper used by both seeded fixtures:

```
def _seed_user(*, is_admin: bool) -> dict:
    """Create a User + session row, return dict with user, session_id, signed_cookie."""
    from app.services.auth import hash_password
    from app.services.sessions import regenerate_session
    from app.signing import sign_session_id
    from app.models.user import User
    from app.main import async_session_factory

    async def _do() -> dict:
        async with async_session_factory() as db:
            uname = f"admin-{uuid.uuid4().hex[:8]}" if is_admin else f"user-{uuid.uuid4().hex[:8]}"
            user = User(
                username=uname,
                email=f"{uname}@example.com",
                password_hash=hash_password("twelve-chars-min-password"),
                is_admin=is_admin,
                is_active=True,
            )
            db.add(user)
            await db.flush()
            session_id = await regenerate_session(db, None, user.id)
            await db.refresh(user)
            return {
                "user": user,
                "session_id": session_id,
                "signed_cookie": sign_session_id(session_id),
            }

    return asyncio.run(_do())
```

Note: `asyncio.run` is acceptable here because the seeded-user fixtures are sync (the tests using them are sync TestClient tests, not asyncio tests). The async_session_factory + the async path are needed because Phase 2's `regenerate_session` is async-only.
