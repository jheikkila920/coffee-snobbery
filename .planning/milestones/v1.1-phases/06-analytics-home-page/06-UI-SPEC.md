## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chip gaps |
| sm | 8px | Compact intra-card element spacing, button icon gap |
| md | 16px | Default element spacing, card internal padding (p-4), stat row gaps, card action link padding |
| lg | 24px | Section heading bottom margin (mb-6), card gap |
| xl | 32px | Page section vertical gap between analytics cards |
| 2xl | 48px | Page top/bottom padding (py-12) |
| 3xl | 64px | n/a (no page-level block this large in this phase) |

**Container:** `mx-auto max-w-6xl px-6 py-12` — matches all existing catalog/sessions pages. Confirmed from `sessions.html`, `coffees.html`.

Exceptions:
- Cold-start empty-state vertical centering uses `py-16` (64px) — matches Phase 5 empty-state convention in `session_list.html`.
- Touch targets for "Log session" / "Brew again" links: `px-4 py-2` = 40px height minimum. Acceptable at the 44px guidance because these are navigation links, not primary form controls. (Phase 11 owns the formal ≥44px audit.)

**Note:** `gap-3` and `px-3` (12px) are not used in this phase. Any component that might naturally use 12px uses `gap-4`/`px-4` (16px) instead, keeping the spacing vocabulary strictly within the declared set {4, 8, 16, 24, 32, 48, 64}.

## Component Inventory

The following components/patterns are used in Phase 6. All are reused from prior phases — no new components introduced.

### Analytics Card Shell
```
<section class="rounded-lg border border-espresso-200 bg-cream-100 p-4 dark:bg-espresso-900 dark:border-espresso-800">
  <h2 class="text-xl font-semibold mb-4">{Card Title}</h2>
  {card body}
</section>
```
- Each card is an independent HTMX lazy-load target
- Cards stack vertically in a single column on mobile (`space-y-4` or `grid grid-cols-1`)
- On desktop (md+), two-column grid layout is permitted for the preference profile cards (four dimensions): `grid grid-cols-1 md:grid-cols-2 gap-4`
- Width: full-width within `max-w-6xl` container

### Stat Row (within a card)
```
<div class="flex items-baseline justify-between gap-4 border-b border-espresso-100 dark:border-espresso-800 py-2">
  <span class="text-base">{label}</span>
  <span class="tabular-nums text-base font-semibold">{value}</span>
</div>
```
Note: `gap-4` (16px) is used here. `gap-3` (12px) is outside the declared spacing scale and is not used.

### Analytics Loading Skeleton
While a card is loading (HTMX request in flight), show a muted placeholder inside the card shell:
```
<div class="animate-pulse space-y-2">
  <div class="h-4 bg-espresso-100 dark:bg-espresso-800 rounded w-3/4"></div>
  <div class="h-4 bg-espresso-100 dark:bg-espresso-800 rounded w-1/2"></div>
</div>
```

### Progress Meter (cold-start)
Single horizontal bar, espresso-700 fill on cream-200 track:
```
<div class="h-2 rounded-full bg-espresso-200 dark:bg-espresso-800 overflow-hidden">
  <div class="h-2 rounded-full bg-espresso-700" style="width: {pct}%"></div>
</div>
```

### Pill / Badge
```
<span class="inline-flex items-center rounded-full bg-espresso-700 px-2 py-1 text-sm text-cream-50">{text}</span>
```
Used for: session counts on top-coffees card, flavor descriptor tags.

Note: `py-1` (4px) is required here. `py-0.5` (2px) violates the 4px spacing grid and is not used anywhere in this phase.

## Page Layout

### Vertical Card Order (mobile-first, confirmed with CONTEXT.md §"Claude's Discretion")

Phase 6 ships this card order. Phase 11 owns nav chrome; Phase 6 templates must work without it.

1. **Page header** — `<h1>Home</h1>` + "Log session" CTA link (always visible, eager)
2. **Recent brews** (HOME-07) — last 10 sessions, eager-loaded (no lazy), edit + brew-again links
3. **Cold-start gate block** — if user has `<3 sessions OR <5 distinct observed flavor notes`:
   - Shows progress meter + copy (D-01/D-02/D-03)
   - Hides items 4–8 entirely
4. **Top coffees** (HOME-01) — lazy, `hx-trigger="load delay:100ms"`
5. **Preference profile** (HOME-02) — lazy, `hx-trigger="load delay:200ms"`, 2-col grid on md+
6. **Top flavor descriptors** (HOME-03) — lazy, `hx-trigger="load delay:300ms"`
7. **Roast freshness buckets** (HOME-04) — lazy, `hx-trigger="load delay:400ms"`
8. **Sweet spots** (HOME-05) — lazy, `hx-trigger="load delay:500ms"`
9. **Unrated coffees** (HOME-08) — lazy, `hx-trigger="load delay:150ms"` (between recent brews and top coffees; low query cost, useful to a new user)

**AI card slot** (HOME-06) — Phase 7. Phase 6 ships no AI card; the slot does not appear. Phase 7 will insert it between Sweet Spots and Unrated coffees (or at top of aggregate block — Phase 7 owns that decision).

**Stagger math:** 50ms base, 50ms increment = 100/150/200/300/400/500ms. Total spread: 500ms max. Within HOME-09 requirement (50–150ms stated for individual spacings; the spec means each card's trigger is N×50ms after the previous, not that every card waits only 50–150ms total).

## HTMX Fragment Endpoints

Each analytics card is a separate fragment endpoint. Follows Phase 4/5 pattern: `Cache-Control: no-store` + `Vary: HX-Request` via `FragmentCacheHeadersMiddleware` (automatic, no per-route config needed).

| Fragment route | hx-trigger | Card |
|---|---|---|
| `GET /home/cards/recent-brews` | eager (included in page render OR `load delay:50ms`) | Recent brews |
| `GET /home/cards/top-coffees` | `load delay:100ms` | Top coffees |
| `GET /home/cards/preference-profile` | `load delay:200ms` | Preference profile |
| `GET /home/cards/flavor-descriptors` | `load delay:300ms` | Flavor descriptors |
| `GET /home/cards/roast-freshness` | `load delay:400ms` | Roast freshness |
| `GET /home/cards/sweet-spots` | `load delay:500ms` | Sweet spots |
| `GET /home/cards/unrated-coffees` | `load delay:150ms` | Unrated coffees |

**Planner note:** Recent brews (HOME-07) is eager-loaded in the initial page render (CONTEXT D-01 — always visible regardless of gate). The HTMX fragment approach is optional for this card; inline include is acceptable.

## Copywriting Contract

### Primary CTA
| Context | Copy |
|---------|------|
| Header CTA | "Log session" |
| Recent brews — empty | See cold-start empty state below |
| Recent brews — session row action | "Brew again" |
| Recent brews — session row action | "Edit" |

**"Edit" label — Phase 5 consistency exception:** The single-word "Edit" CTA in Recent Brews rows is an intentional match to the label used in Phase 5's `session_row.html`. The executor must not substitute "Edit session", "Modify", or any other variant. Consistency with the existing session list is the rationale; this overrides the general preference for verb + noun CTAs.

### Cold-Start Empty State (gate not cleared)
Shown when `sessions < 3 OR distinct_observed_flavor_notes < 5`.

**Heading:** "Build your taste profile."

**Progress meter label:** Dynamic — computed from live counts:
- If `sessions < 3` AND `notes < 5`: "Log {3 - sessions} more brew{s} and add {5 - notes} more flavor note{s} to unlock insights."
- If `sessions >= 3` but `notes < 5`: "Add {5 - notes} more flavor note{s} to your brews to unlock insights."
- If `notes >= 5` but `sessions < 3`: "Log {3 - sessions} more brew{s} to unlock insights."
- (Exact ROADMAP example: "Log 2 more brews and add 3 more flavor notes to unlock recommendations.")

**Body below meter:** "In the meantime, your recent brews and catalog are below."

**CTA:** "Log session" (links to `/brew/new`)

### Aggregate Card — Sparse State (gate cleared, but card query returns no qualifying rows)
Per decision D-04: every card renders with a hint. Per D-05: distinguish all-unrated cause.

| Card | Generic hint (sparse data) | All-unrated nudge (D-05) |
|------|---------------------------|--------------------------|
| Top coffees | "No coffees with 2+ sessions yet — keep logging." | "Rate some brews to see your top coffees." |
| Preference profile | "Not enough sessions per dimension yet — keep logging." | "Rate some brews to see your preference profile." |
| Flavor descriptors | "No flavor notes in 4.0+ rated sessions yet." | "Rate some brews 4.0+ and add observed flavor notes to see top descriptors." |
| Roast freshness | "No sessions with a bag roast date yet — add roast dates to your bags." | n/a (freshness doesn't require ratings) |
| Sweet spots | "Not enough sessions per combination yet (need 3 per match)." | "Rate some brews to find your sweet spots." |

### Aggregate Card — AI Prose Placeholder (Phase 7 slot)
Not rendered in Phase 6. The sweet-spots card ends after the SQL-derived table. No "AI insights coming soon" placeholder — clean omission until Phase 7 wires the card.

### Error State (card fragment fails to load)
If a fragment endpoint returns a 5xx or HTMX request fails:

**Copy:** "Couldn't load this section. Refresh the page to try again."

This is rendered inside the card shell so the card's heading is still visible. No inline retry button in Phase 6 (manual page refresh is sufficient; auto-retry is Phase 7 concern for the AI card only).

### Unrated Coffees Card (HOME-08)
**Section heading:** "Not tried yet"

**When empty:** "You've brewed every coffee in the catalog."

**Row format:** Coffee name + roaster + "Log session" link to `/brew/new?coffee_id={id}`

### Recent Brews Card (HOME-07)
**Section heading:** "Recent brews"

**When zero sessions:** "No brews logged yet. The snobbery awaits." (matches Phase 5 `session_list.html` — consistency enforced)

**"Edit" link label:** "Edit" (matches Phase 5 session_row.html — deliberate single-word exception; see Phase 5 consistency exception note above)

**"Brew again" link label:** "Brew again" (matches Phase 5 session_row.html)

### Destructive Actions
None. Phase 6 is a read-only analytics surface. No deletes, no confirms required.

## Interaction Contract

### Lazy Loading
- Each analytics card div includes `hx-get="{fragment_url}" hx-trigger="load delay:Nms" hx-swap="innerHTML"` on the card shell's inner content area.
- The card shell (border, heading) renders immediately in the initial page response.
- The card body shows the loading skeleton until the fragment response arrives.
- No global loading indicator — each card resolves independently.

### "Brew again" from Recent Brews
- Links to `/brew/new?from={session_id}` — identical to Phase 5 convention.
- No HTMX swap; full navigation.

### "Log session" CTA
- Plain anchor `<a href="/brew/new">` — no HTMX.

### Progress Meter
- Static server-rendered (no Alpine). Width is a server-computed inline style percentage.
- No animation on first render. Tailwind `transition-all duration-500` may be added but is not required.

### Card Action Links (Edit / Brew again)
- Padding: `px-4 py-2` — 16px horizontal, 8px vertical. Uses declared md (16px) and sm (8px) tokens.
- `px-3` (12px) is outside the declared spacing scale and is not used on any action link in this phase.

### No Destructive Interactions
Phase 6 ships zero delete/archive/disable actions. No modals, no confirmation dialogs.

## Accessibility Contract

- Each `<section>` analytics card carries `aria-labelledby="{card-heading-id}"`.
- Rating stars rendered as `★` (Unicode) use `aria-hidden="true"` on the decorative character; the numeric value follows immediately for screen readers — matches Phase 5 session_row.html.
- Progress meter uses `role="progressbar" aria-valuenow="{pct}" aria-valuemin="0" aria-valuemax="100" aria-label="Profile completion"`.
- All table `<th>` cells carry `scope="col"`.
- Links within cards carry descriptive `aria-label` when the visible text alone is ambiguous (e.g. "Log session with {coffee_name}").

## Mobile-First Constraints

- All analytics cards stack as a single column at 375px. No horizontal scroll.
- Card touch targets (Edit / Brew again buttons): `px-4 py-2 text-sm` button size — acceptable for action links within dense list rows (Phase 11 owns the formal 44px audit).
- "Log session" header CTA: `px-4 py-2 text-base` = full-size button, ≥44px height.
- At `md:` breakpoint (768px+), preference profile (HOME-02) renders as a 2-column grid (`grid-cols-2`) since it has 4 dimensions (origin, process, roaster, roast level) that read better side-by-side.
- All other cards: single column at all breakpoints.
